Обработка "Формирование заказа на основании продаж"

Назначение

Предназначена для автоматического заполнения заказа поставщика на основании анализа продаж товара поставщику за период.

Формы

  • Форма. Основная форма настройки обработки заполнения заказа поставщику на основании продаж.

Поля

  • Подразделение. Подразделение, для которого находится цена закупки и границы минимального и максимального запаса.
  • Поставщик. Поставщик, оформляемого заказа.
  • Дата поставки. Дата поставки указанная в документе Заказ поставщику.
  • Дата второй поставки. Предполагаемая дата поставки следующего заказа у этого поставщика.
  • Период анализа продаж. Период, из которого будут взяты данные о продажах товара поставщика для получения среднего количества продаж товара в день.
  • Тип цен закупки. Тип цен для получения цены закупи товара у поставщика и минимальной цены в целом по всем поставщикам. Данные берутся из регистра «Цены».
  • Способ округления. Метод округления рекомендуемого количества заказа.
  • Учитывать отсутствие на складе. Включенный флаг означает, что при нахождении среднего количества продаж товара в день будут исключены дни, в которые товар отсутствовал на складе.

Товары

  • Номенклатура. Товар, который продавался за анализируемый период и поставлялся данным поставщиком.
  • Остаток. Текущий запас товара на дату установленную в документе заказ поставщику, из которого была запущена обработка заполнения.
  • Оборот за период. Количество проданного товара за период анализа продаж.
  • Оборот в день. Среднее количество продаж за период анализа продаж.
  • Уже заказано. Количество, которое уже заказано поставщикам и прибудет раньше, чем вторая поставка.
  • Дата поступления. Самая поздняя дата поступления уже заказанного товара у поставщиков. Дата не может быть больше, чем дата второй поставки.
  • Количество. Рекомендуемое количество заказа у поставщика. Значение из колонки "Уже заказано" не влияет на расчет рекомендуемого заказа.
  • Закупочная цена. Цена закупки у поставщика по выбранному типу цен из регистра «Цены».
  • Цена продажи. Цена, по которой продали товар.
  • Минимальная цена закупки. Минимальная цена закупки товара по выбранному типу цен из регистра «Цены».
  • Минимальный запас. Граница минимального запаса.
  • Максимальный запас. Граница максимального запаса.

Описание метода

Подход прогнозирования расхода запаса товара на складе за период между датой оформляемого заказа и датой поставки следующего заказа поставщику основан на среднем скользящем прогнозе продаж. Результат представляет собой простое среднее арифметическое значение продаж товара за один день найденное из данных о продажах за период. Если для товара есть ограничения минимального и максимального запаса, то количество заказа будет подкорректировано, чтоб полученный запас при поставке не превысил максимальное ограничение и остаток запаса, до второй поставки заказанного был не ниже минимального порога запаса.

Способ вызова

Вызывается из подменю "Заполнение" в документе Заказ поставщику

Для текстового файла разделителем считаем символ Tab, для csv - “;”. 

При этом структура файла должна быть Артикул, Цена, (Наименование, Производитель). Артикул и Цена - обязательные поля, Наименование и Производитель могут присутствовать, могут не присутствовать. Наименование нужно в случае создания номенклатуры по файлу. Производитель указывается в настройках, поэтому если где-то в файле он не будет указан, то все равно поиск номенклатуры в базе будет осуществляться по связке “Артикул + Производитель”. Производитель должен быть занесен в базу, поиск идет по наименованию.

Необходимо указать подразделение документа - оно будет подставлено в соответствующий реквизит документа “Изменение цен”. И именно по нему будет определяться текущая цена номенклатуры, если будет снята галка “Добавлять, если цена не изменилась”. 

Указывается также дата начала действия цен и подчиненные подразделения. Все эти реквизиты соответствуют реквизитам документа “Изменение цен”. 

Выбираем один или несколько типов цен, в которые будет загружаться документ.  При этом следует учесть, что чем больше выбрано типов цен и подчиненных подразделений, тем больше документ будет создавать движений по регистру цен.  

На закладке параметров загрузки есть возможность задать максимальное количество строк в одном документе:

То есть для очень больших файлов или прайс-листов возможно создание нескольких документов. Если в файле 150 строк, а мы указали строк в документе 120, то будет создано 2 документа: 120 и 30 строк. После создания эти документы можно будет сразу открыть для пользователя, а также провести (галки “Открывать созданные документы” и “Проводить созданные документы”).

Флажок “Добавлять, если цена не изменилась”, если он установлен, говорит о том, что программа не проверяет текущие цены на номенклатуру в базе. Т.е. грузит данные как есть. В случае, если флажок снят, запускается механизм поиска цен на номенклатуру в базе, по выбранному подразделению документа, на дату начала действия. Если цена найдена и она равна цене прайса или цене из файла, для  выбранного типа цен, то по данной номенклатуре и типу цен строка не будет добавлена. 

Кроме того, при создании документа из файла, есть возможность автоматически создать номенклатуру. Для загрузки из прайс-листа такой возможности нет. 

Для загрузки ставим флажок “Создавать номенклатуру”, появляются поля для заполнения типа создаваемой номенклатуры и вида номенклатуры. При нажатии “Загрузить цены” - программа вначале создаст номенклатуру, а затем уже для данной номенклатуры создаст документы изменения цен. 

В документы изменения цен добавляется только номенклатура с видом: Товар; Комплект; Тара; Шины; Диски; Номерные агрегаты; ЛКМ. Если же при заполнении документа по артикулу будет найдена номенклатура с другим типом (например, Услуга) - она не будет добавлена в документ изменения цен.


Загрузка из прайс-листа. 

При варианте загрузки “Из прайс-листа” необходимо выбрать прайс-лист контрагента. При этом существует 2 вида прайс-листов: которые хранятся в базе, в регистре сведений “Прайс-листы контрагентов” и которые хранятся на внешних источниках.

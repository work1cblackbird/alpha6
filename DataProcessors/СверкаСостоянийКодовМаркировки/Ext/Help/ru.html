Обработка "Сверка состояний кодов маркировки"

Назначение

Служит для сверки состояний кодов маркировки (КМ) по одной организации (организации которой принадлежит сертификат) текущей базы данных и данных из "Честного знака" (ЧЗ). Для корректного сопоставления данных необходимо заполнение регистра "Штрихкоды", где происходит привязка GTIN к номенклатуре, обязательным является активация флага GTIN. В таблице выделяются салатовым цветом данные, где совпали сопоставленные состояния КМ. 

Обработка выгружает данные за указанный период, если период выгрузки не задан, выгружаются документы за всю работу текущей базы.

Отборы

  • Эмитирован с ... по. Реквизиты являются не обязательными для заполнения. При отсутствии периода выгружается срез данных из ЧЗ и текущей базы для последующей сверки. При заполненном значении происходит отбор по дате эмиссии соответственно.
  • Организация. Реквизит является обязательным для заполнения. При смене отбора по организации произойдет смена сертификата, подключенного к базе данных.
  • Отбор GTIN. Реквизит является не обязательным для заполнения. Добавляет отбор по GTIN. 
  • Выводить КМ переданные другому собственнику. По умолчанию отключен. При получении КМ из ЧЗ с активным флагом данные получены не будут т.к. в ЧЗ срез текущих данных по КМ (при передаче другому собственнику в ЧЗ изменяется владелец).

Основная табличная часть «Таблица сверки»


  • Артикул. Реквизит выводится если произошло сопоставление или в базе есть документы с участием номенклатуры при указании КМ. 
  • Номенклатура. Реквизит выводится если произошло сопоставление или в базе есть документы с участием номенклатуры при указании КМ.  
  • Характеристика. Реквизит выводится если произошло сопоставление или в базе есть документы с участием номенклатуры при указании КМ. 
  • GTIN. Реквизит выводится если произошло сопоставление или в базе есть документы с участием номенклатуры при указании КМ.  
  • Наименование ЧЗ. Реквизит выводится если произошло сопоставление или в ЧЗ есть данные по номенклатуре и КМ. 
  • КМ 1С. Реквизит выводится если произошло сопоставление или в базе есть документы с участием номенклатуры при указании КМ.  
  • Состояние 1С. Текущее состояние КМ из базы. Реквизит выводится если произошло сопоставление или в базе есть документы с участием номенклатуры при указании КМ. 
  • Дата состояния 1С. Дата последнего состояния КМ в текущей базе. Реквизит выводится если произошло сопоставление или в базе есть документы с участием номенклатуры при указании КМ.  
  • КМ ЧЗ. Реквизит выводится если произошло сопоставление или в ЧЗ есть данные по номенклатуре и КМ.
  • Состояние ЧЗ. Текущее состояние КМ из ЧЗ. Реквизит выводится если произошло сопоставление или в ЧЗ есть данные по номенклатуре и КМ. 
  • Дата состояния ЧЗ. Дата последнего изменения состояния КМ. Реквизит выводится если произошло сопоставление или в ЧЗ есть данные по номенклатуре и КМ.
  • Расхождение. ИСТИНА если состояния равны.

Дополнительная табличная часть «Данные учета»


  • На складах. Текущий остаток (все склады) по выбранной номенклатуре, выводится при условии, что в строке таблицы сверки заполнено поле "Номенклатура".  
  • В производстве. Текущий остаток (в производстве) по выбранной номенклатуре, выводится при условии, что в строке таблицы сверки заполнено поле "Номенклатура". 
  • Общий остаток. Сумма остатка в производстве и остатка на складах.  
  • Количество КМ в 1С. Количество КМ (в текущей базе) по выбранной номенклатуре, выводится при условии, что в строке таблицы сверки заполнено поле "Номенклатура" . 
  • Количество КМ в ЧЗ. Количество КМ (из ЧЗ) по выбранной номенклатуре, выводится при условии, что в строке таблицы сверки заполнено поле "Наименование ЧЗ".

Механизм обмена данными. Обмен данными Альфа-Авто: Автосалон+Автосервис+Автозапчасти КОРП. Редакция 6 - Бухгалтерия.

 

План обмена предназначен для хранения изменений и фильтрации данных при обмене Альфа-Авто: Автосалон+Автосервис+Автозапчасти КОРП. Редакция 6 - Бухгалтерия.

Имеется возможность фильтровать данные при обмене по организациям.

Префикс загружаемых документов определяет добавляемую часть к номеру объектов, которые загружаются при обмене. Префикс добавляется только для документов, дата которых не меньше чем начальная дата обмена документами. Начальная дата так же определяет фильтрацию выгружаемых документов. Выгрузке подлежат только те документы, дата которых не меньше указанной.

Правила отправки данных

Флаг «Переносить наименование номенклатуры как Наименование [Артикул]». При сброшенном флажке, номенклатура у которой нет соответствия загружается следующим образом: номенклатура ищется по наименованию, если элемент справочника с таким же именем не найден, создается новый элемент, если найден, устанавливается соответствие загружаемого элемента и найденного. При включенном флажке, номенклатура у которой нет соответствия загружается следующим образом: наименование номенклатуры принимается за конкатенированную строку из наименования и артикула в квадратных скобках (пример: Петля двери задней верхняя MMC [MB546471]). Номенклатура ищется по наименованию по всему справочнику, если элемент справочника с таким же именем не найден, создается новый элемент, если найден, устанавливается соответствие загружаемого элемента и найденного.

Документ "Банковская выписка"

Для корректной отправки в справочнике «Статьи движения денежных средств» необходимо заполнить реквизит «Операция». В предопределенных элементах справочника этот реквизит заполнится автоматически. Для созданных в ручную статей, заполнить операцию нужно самостоятельно.

Справочник "Типы номенклатуры"

Справочник "Типы номенклатуры" выгружается в бухгалтерию в справочник "Виды номенклатуры". Выгрузка типов номенклатуры происходит в три вида: Товар, Услуга, Тара. При необходимости в настройках узла можно расширить передаваемые типы номенклатуры. По ссылке "Установить соответствие типов номенклатуры " добавляем в табличную часть Тип номенклатуры и Наименование вида из бухгалтерии. При заполнении таблицы соответствия передача типа номенклатуры проходит только по наименованию указанному в графе Наименование, идентификатор типа номенклатуры не передается. В бухгалтерии при получении, сопоставление проходит по наименованию, сопоставления и связки по идентификаторам нет так как в этом случае он не передается.

В случае если тип номенклатуры не указан в таблице соответствия или таблица соответствия не заполнена типы номенклатуры передаются в три вида: Товар, Услуга, Тара.

Документ "Банковская выписка"

В настройках синхронизации данных на закладке Правила получения данных добавлена таблица «Установить соответствие операций и статей». При обновлении конфигурации и добавлении нового узла в ней будут заполнен существующий в конфигурации перечень видов операций и предопределенные статьи движения денежных средств. При обновлении таблица соответствий заполняется не полностью. Для полного заполнения нужно создать статьи движения денежных средств с нужным видом операции.

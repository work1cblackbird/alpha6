Справочник "Интерфейсы для внешних систем"

Назначение

Предназначен для хранения параметров взаимодействия с внешними системами (банковский эквайринг, облачные кассы и т.д.), построенными на основе http-сервисов

Способ вызова

Вызывается на панели Администрирование командой Интерфейсы для внешних систем
Команда на панели появляется при включении (см. Администрирование --> Настройка параметров учета) хотя-бы одного из следующих разделов учета:
  • Онлайн-оплаты (доступен начиная с данной версии)
  • Облачные кассы (появится в следующих версиях)

Реквизиты

Общие для всех типов интерфейсов реквизиты:
  • Тип интерфейса. Выбирается из списка. В данной версии доступны значения:
    • 1С (платежный шлюз СБП)
    • ЮKassa (платежный шлюз)
  • Назначение. Определяет структурные единицы компании, для которых данный интерфейс доступен. Возможные значения:
    • Ничего не выбрано - интерфейс доступен для всей компании
    • Выбрана организация - интерфейс доступен только для выбранной организации
    • Выбран филиал - интерфейс доступен только для выбранного филиала
    • Выбрано подразделение компании - интерфейс доступен только для выбранного подразделения
  • Наименование. Формируется автоматически в зависимости от типа интерфейса и назначения. При необходимости можно изменить
  • Адрес. Содержит адрес (URL хоста) сервиса. Заполняется значением по умолчанию при выборе типа интерфейса
  • Способ авторизации. Выбирается из списка. Возможные значения:
    • Логин и пароль - авторизация на сервисе выполняется через логин и пароль
    • Токен - авторизация на сервисе выполняется через токен
                  При создании нового элемента справочника способ авторизации устанавливается в "Логин и пароль"
  • Логин. Строка с именем авторизованного пользователя Виден на форме, если выбран способ авторизации "Логин и пароль"
  • Пароль. Строка пароля. Виден на форме, если выбран способ авторизации "Логин и пароль"
  • Токен. Строка токена авторизации. Виден на форме, если выбран способ авторизации "Токен"
  • Использовать защищенное соединение. Данный флажок должен быть установлен для https-сервисов и снят для http-сервисов
  • Порт. Адрес порта. Для https-сервисов значение по умолчанию 443, а для http-сервисов значение по умолчанию 80.
Состав реквизитов группы "Дополнительные параметры" на форме элемента определяется выбранным типом интерфейса. Ниже приведен полный список этих реквизитов в алфавитном порядке:
  • Банк-участник СБП. Идентификатор банка-участника системы быстрых платежей. Доступные для выбора из списка значения:
    • PayMaster (ООО "Пэймастер") - при выборе этого значения используется способ авторизации "Токен"
    • WebMoney (Банк ККБ) - при выборе этого значения используется способ авторизации "Токен"
    • Банк "Открытие" - при выборе этого значения используется способ авторизации "Логин и пароль"
    • Филиал "Дело" ПАО "СКБ-банк" - при выборе этого значения используется способ авторизации "Логин и пароль"
  • Время жизни платежной ссылки в минутах. Время, в течение которого возможна оплата счета клиентом в режиме онлайн. Отсчет времени начинается с момента получения платежной ссылки в сервисе. Устанавливается в 60 минут. Менять его не рекомендуется
  • Идентификатор мерчанта. Идентификатор продавца. Его назначает эквайер при заключении с ним договора на использование сервиса онлайн-оплаты
  • Касса ККМ. Касса, связанная с данным сервисом
  • Обрабатывать входящие уведомления (callback). Установка флажка означает, что при взаимодействии с сервисом будут обрабатываться приходящие от него уведомления. Для обработки входящих уведомлений нужен собственный https-сервис
  • Сайт после оплаты. Адрес сайта, на который будет переведен клиент после онлайн-оплаты. Обычно это - сайт компании. Значение по умолчанию - "https://yandex.ru"
  • Создавать фискальные чеки в данном сервисе. Установка флажка означает, что создание и отправка фискальных чеков происходят в сервисе эквайера. Такая возможность определяется в договоре эквайринга, но в данной версии еще не реализована
  • Фискальный регистратор. ФР, связанный с данным сервисом
  • Шаблон сообщений SMS (Счет на оплату) Шаблон, используемый для генерации текстов сообщений SMS с платежной ссылкой для онлайн-оплаты документов Счет на оплату. Если значение не выбрано, то сообщения SMS будут генерироваться программно
  • Шаблон сообщений SMS (Счет на оплату за автомобили). Шаблон, используемый для генерации текстов сообщений SMS с платежной ссылкой для онлайн-оплаты документов Счет на оплату за автомобили. Если значение не выбрано, то сообщения SMS будут генерироваться программно
  • Шаблон электронных писем (Счет на оплату). Шаблон, используемый для генерации текстов электронных писем с платежной ссылкой для онлайн-оплаты документов Счет на оплату. Если значение не выбрано, то тексты писем будут генерироваться программно
  • Шаблон электронных писем (Счет на оплату за автомобили). Шаблон, используемый для генерации текстов электронных писем с платежной ссылкой для онлайн-оплаты документов Счет на оплату за автомобили. Если значение не выбрано, то тексты писем будут генерироваться программно

Справочник "Виды планов компании"

Назначение

 

Это корневой объект ядра, этот иерархический список содержит описания всех используемых в компании видов и вариантов документов планирования. Состав и содержание элементов этого справочника и определяют структуру блока поддержки бизнес-логики. Любой элемент этого справочника, будучи единожды настроенным, используется на протяжении всего периода эксплуатации БД. При необходимости модификации поведения вида планов просто создается новый элемент, обеспечивая, таким образом, регулярность и соответствие настройкам всех хранимых данных предыдущих периодов.

Предопределенные элементы

  • План продаж по товарным категориям месячный.
  • План продаж по номенклатуре в пределах категории месячный.
  • План закупок по номенклатуре в пределах категории месячный.
  • План закупок по номенклатуре в пределах категории по поставщикам месячный.
  • План доходов и расходов.
  • План движения ДС.

Параметры

Длина кода: 9, тип кода:Строка.

Длина наименования: 100.

Иерархический.

Реквизиты

  • Вид объекта планирования. Ссылается на план вида характеристик «Аналитические разрезы планирования». Используется для фильтрации состава аналитических объектов в обработках заполнения.
  • Периодичность планирования. Реквизит является обязательным для заполнения. Ссылается на перечисление «ПериодичностьАнализаНакопительныхСкидок». Периодичность заполнения и сверки плановых и фактических документов.
  • Вид аналитики планирования 1. Реквизит является обязательным для заполнения. Ссылается на план вида характеристик «Аналитические разрезы планирования». Тип первого аналитического реквизита таблицы план-фактных документов.
  • Вид аналитики планирования 2. Ссылается на план вида характеристик «Аналитические разрезы планирования». Тип второго аналитического реквизита таблицы план-фактных документов.
  • Показатель плана 1. Тип значения: «Строка». Реквизит является обязательным для заполнения. Наименование первого числового показателя документов планирования.
  • Показатель плана 2. Тип значения: «Строка». Наименование второго числового показателя документов планирования.
  • Показатель плана 3. Тип значения: «Строка». Наименование третьего числового показателя документов планирования.

Табличная часть «Параметры плана»

  • Вид параметра. Ссылается на план вида характеристики «Параметры планирования» Параметр планового документа.
  • Тип значения. Тип значения: «Строка». Тип значения параметра планового документа.

Табличная часть «Базовые планы»

  • Вид плана. Ссылается на справочник «Виды планов компании». Вид плана, от которого зависит данный.
  • Вид зависимости. Ссылается на перечисление «ВидыЗависимостейПланов». По своему подразделению, по вложенным подразделениям, по родительскому подразделению.
  • Объект планирования по аналитике. Тип значения: «Строка». Объект планирования не связан с базовым по объекту планирования, или номер аналитики базового плана.

Табличная часть «Подразделения»

  • Подразделение. Ссылается на справочник «Подразделения компании». Подразделение, по которому ведётся планирование.
  • Отсрочка плана. Тип значения: «Число». Количество дней отсрочки плана от начала периода планирования.
  • Отсрочка факта. Тип значения: «Число». Количество дней отсрочки факта от конца периода планирования для расчета факта.

Табличная часть «Отчеты и обработки»

  • Вид объекта. Тип значения: «Строка». Выпадающий список, состоящий из:
    • встроенный запрос из текстового макета;
    • внешний запрос из текстового файла;
    • встроенная обработка;
    • внешняя обработка.
  • Путь к объекту. Тип значения: «Строка». Имя объекта метаданных или путь к файлу объекта.
  • Флаг заполнения плана. Тип значения: «Булево». Признак наличия процедуры заполнения планового документа по установленным параметрам.
  • Флаг проверки плана. Тип значения: «Булево». Признак наличия процедуры проверки планового документа по установленным параметрам.
  • Флаг заполнения факта. Тип значения: «Булево». Признак наличия процедуры заполнения фактического документа по установленным параметрам.
  • Наименование. Тип значения: «Строка». Реквизит является обязательным для заполнения. Наименование отчета / обработки для интерактивного выбора.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов.
  • Форма группы. Основная форма редактирования группы справочника.

Способ вызова

Вызывается из меню "Справочники --> Планирование --> Виды планов компании".

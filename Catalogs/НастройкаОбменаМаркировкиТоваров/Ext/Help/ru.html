Справочник "Настройка обмена с личным кабинетом"

Назначение

В данном справочнике указываются настройки подключения через API к личному кабинету Государственной информационной системы мониторинга за оборотом товаров, подлежащих обязательной маркировке средствами идентификации (ГИС МТ). Для каждой организации указывается своя настройка. 

Параметры

Длина кода: 9, тип кода: Строка.

Длина наименования: 25.

Не иерархический.

Реквизиты

  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Организация, к которой относится настройка подключения.
  • Адрес. Реквизит является обязательным для заполнения. Тип значения: «Строка». Базовый адрес контура ГИС МТ. 
  • Порт. Реквизит является обязательным для заполнения. Тип значения: «Число». Порт контура ГИС МТ.
  • Версия формата обмена. Реквизит является обязательным для заполнения. Ссылается на перечисление «Версии форматов обмена Честного знака». Указывается текущая версия обмена с ГИС МТ. 
  • Сертификат. Ссылается на справочник: «Сертификаты ключей электронной подписи и шифрования». Ссылка на сертификат, который используется для аутентификации в ГИС МТ. Если не заполнен сертификат, тогда перед отправкой запроса в Честный знак необходимо дополнительно подписывать данные аутентификации.
  • Защищенное соединение. Тип значения: «Булево». Признак того, что при отправке запроса и используется защищенное соединение. По умолчанию данный реквизит устанавливается. 
  • Время ожидания. Реквизит является обязательным для заполнения. Тип значения: «Число». Время ожидания ответа от ГИС МТ при отправке запроса. Указывается в секундах.
  • Адрес СУЗ. Реквизит является обязательным для заполнения. Тип значения: «Строка». Базовый адрес контура API станции управления заказами (СУЗ). 
  • Порт СУЗ. Реквизит является обязательным для заполнения. Тип значения: «Число». Порт контура API СУЗ. 
  • OMS ID. Реквизит является обязательным для заполнения. Тип значения: «Строка». Идентификатор СУЗ, доступен в настройках СУЗ.
  • Использовать True API. Тип значения: «Булево». Для обмена с Честным знаком использовать методы True API. С 04.09.2023 года прекращается обмен методами API ГИС МТ.
  • Токен для ККТ. Тип значения: «Строка». Токен применяется для проверок продукции на кассе перед продажей для разрешительного режима. Действует до 1 марта 2025 года.

Табличная часть «Список устройств»

  • Подразделение. Ссылается на справочник «Подразделения компании». Используется для указания подразделению конкретного токена. Если не заполнен, то данный токен используется для всей организации по умолчанию. 
  • Идентификатор соединения. Тип значения: «Строка». Необходим для формирования клиентского токена, отправляемого клиентом в заголовке HTTP-запроса. Доступен в настройках СУЗ из списка устройств.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов. 
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов. 

Способ вызова

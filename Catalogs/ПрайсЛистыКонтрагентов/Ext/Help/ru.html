Справочник "Прайс-листы контрагентов"

Назначение

Справочник предназначен для хранения данных о товарах контрагентов, ценах, наличии, сроках поставки. Файлы обмена с прайс-листами могут представлять собой различные форматы: базы данных, обращение к которым может осуществляться посредством ODBC или файлы в формате Microsoft Excel.

Относится к подсистеме: «Снабжение».

Параметры

Длина кода: 8, тип кода: Строка.

Длина наименования: 100.

Не иерархический.

Владельцы справочника: «Контрагенты и контакты».

Реквизиты

  • Владелец. Ссылается на справочник «Контрагенты и контакты». Контрагент, который является владельцем прайс-листа. Информация о прайс-листах сохраняется без учета истории изменения. Для просмотра установленных значений используется список регистра сведений «Прайс-листы контрагентов».
    Вызов: подменю формы "Перейти --> Сведения компании".
  • Валюта прайс-листа. Реквизит является обязательным для заполнения. Ссылается на справочник «Валюты». Валюта, в которой указаны цены товаров данного контрагента.
  • Кодировка. Реквизит является обязательным для заполнения. Тип значения: «Строка». Кодовая страница прайс-листа. Выбирается из выпадающего списка: CP 1251, KOI 8, CP 866, ISO 8859.
  • Размещение. Тип значения: «Булево». Флаг, указывающий в каком формате представлен прайс-лист (файл или источник данных).
  • Подключение. Тип значения: «Строка». Имя файла с данными прайс-листа или строка подключения к источнику данных ODBC.
  • Разделитель. Тип значения: «Строка». Разделитель текстового файла. Поле активно при выборе вида размещения "Файл".
  • Со строки. Тип значения: «Число». Номер первой строки прайс-листа в файле для обработки. Поле активно при выборе вида размещения "Файл".
  • По строку. Тип значения: «Число». Номер последней строки прайс-листа в файле для обработки. Поле активно при выборе вида размещения "Файл".
  • Имя таблицы. Тип значения: «Строка». Имя таблицы, из которой будет осуществляться загрузка прайс-листа при подключении через ODBC. Поле активно при выборе вида размещения "Источник данных".

Табличная часть «Штрихкод»

  • Реквизит справочника. Тип значения: «Строка». Реквизит справочника, который необходимо заполнить из файла.
  • Поле таблицы. Тип значения: «Строка». Номер колонки соответствующей реквизиту справочника. если номер не указан, то соответствующий реквизит справочника заполнятся не будет.
  • Ключ. Тип значения: «Булево». Признак ключевого поля, по этому полю проводится идентификация.

Вкладка «Ценообразование»

  • Установка цены при создании номенклатуры из прайс-листа. Тип значения: «Булево». Если флаг включен, то при создании номенклатуры из прайс-листа будет автоматически создан документ Установка цен компании.
  • Тип цен. Ссылается на справочник Типы цен. Тип цен, для которого будет формироваться документ Установка цен компании при включенном флаге "Установка цены при создании номенклатуры из прайс-листа".
  • Подразделение цены. Ссылается на справочник Подразделения компании. Подразделение, для которого будет устанавливаться цена при автоматическом создании документа Установка цен компании.
  • Цена включает НДС. Тип значения «Булево». Описание цены. указанной в прайс-листе. Если флаг установлен, то считается, что цена в прайс-листе указана с НДС. Если флаг не установлен, то считается что цена указана в прайс-листе без НДС.
  • Ставка НДС. Ссылается на справочник Ставки НДС. Описание цены, указанной в прайс-листе. Если установлен флаг Цена включает НДС, то указывается ставка НДС для цен прайс-листа.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов.

       На формах присутствуют ссылки на элемент справочника «Контрагенты и контакты».

Способ вызова

Вызывается из меню "Справочники -->Идентификационные--> Прайс-листы контрагентов".


Внимание!

Если не установить флажок "Загружать без количества" и не указать соответствие реквизита "Количество" и структуры прайс-листа, либо не указать значение по умолчанию для поля "Количество", то прайс-лист не будет загружен. Для того, чтобы прайс-лист был загружен, необходимо настроить соответствие или установить флажок "Загружать без количества" или установить для реквизита "Количество" значение по умолчанию.

Дополнительная информация

В форме списка и в форме элемента имеется возможность загрузки прайс-листа из внешнего источника в ИБ и возможность очистки прайс-листа.

Форматы данных загрузки прайс-листа следующие:

  • Для случая загрузки из файла MS Excel (.XLS): в колонке "Поле таблицы" цифрами указываются соответствующие реквизитам колонки документа MS Excel, в поле "Со строки" указывается номер строки таблицы MS Excel, с которой начинаются данные, в поле "По строку" указывается номер последней строки с данными. Для приведенного примера:

      1 2 3 4 5 6 7
    1              
    2   6475647364 DVD плеер "Sony" 12000.00 100 1  
    3   1375647344 Плазменный телевизор "Самсунг" 21' 18000.00 10 1  
    4   5474644364 Акустическая система 5.1 "Sven" 9000.00 5 1  
    5              
     
    значения будут такими:
    - "Артикул номенклатуры" = 2;
    - "Наименование номенклатуры" = 3;
    - "Цена номенклатурной позиции" = 4;
    - "Количество номенклатуры" = 5;
    - "Срок поставки номенклатуры" = 6;
    В поле ввода "Со строки" должно быть введено "2", в поле "По строку" - "4".
  • Для случая загрузки из текстового файла (.TXT): данные располагаются в файле через разделитель, указанный в поле ввода "Разделитель". В колонке "Поле таблицы" цифрами указываются номера значений (т.е. количество разделителей с начала строки до требуемого значения плюс 1). В поле "Со строки" указывается начальная строка, а в поле "По строку" - конечная строка текстового файла, которую нужно обработать. Например, для файла, содержащего:

    3647564736,DVD плеер "Sony",12000.00,10,1
    1375647344,Плазменный телевизор "Самсунг" 21',18000.00,10,1
    5474644364,Акустическая система 5.1 "Sven",9000.00,5,1

    значения будут такими:
    - "Артикул номенклатуры" = 1;
    - "Наименование номенклатуры" = 2;
    - "Цена номенклатурной позиции" = 3;
    - "Количество номенклатуры" = 4;
    - "Срок поставки номенклатуры" = 5;
    В поле ввода "Со строки" должно быть введено "1", в поле "По строку" - "3". Поле "Разделитель" должно содержать ",".
  • Для случая загрузки с источника данных ADO в колонке "Поле таблицы" указываются названия полей таблицы базы данных соответствующие реквизитам.

Если поле "Со строки" не заполнено, подразумевается, что там введена цифра 1, если поле "По строку" не заполнено, считываются данные с позиции, указанной в поле "Со строки" и до конца.

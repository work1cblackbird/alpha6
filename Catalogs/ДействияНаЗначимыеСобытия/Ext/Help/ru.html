Справочник "Действие на значимое событие"

Назначение

Справочник "Действия на значимое событие" описывает выполняемые действия при обработке событий от прикладных объектов конфигурации. Каждый элемент справочника описывает одно действие. Предусмотрены следующие варианты действий:

  1. Создать и записать объект. В этом случае необходимо указать тип создаваемого объекта. Создавать можно: объекты ссылочного типа, запись регистра сведений (одну), установить значение константы. Если возможно, то будет задействован механизм ввода создаваемого объекта на основании объекта источника события. Если создается документ, то установкой соответствующего флага, можно выполнить его проведение (если проведение данного документа предусмотрено в конфигурации). Далее необходимо установить правила заполнения реквизитов создаваемого объекта. Реализованы три вида правил заполнения: 1 - указанием точного значения реквизита; 2 - "сослаться" на реквизит объекта источника события (только если объект-источник события имеет ссылочный тип); 3 - заполнить реквизит произвольным кодом на языке 1С.
  2. Создать напоминание. Создается напоминание. Фактически происходит создание записей регистра сведений "Напоминания".
  3. Создать запись в журнале регистрации.
  4. Написать электронное письмо. Создается электронное письмо и, по-необходимости, производится попытка немедленной его отправки.
  5. Написать сообщение SMS. Создается сообщение SMS и, по-необходимости, производится попытка немедленной его отправки.

Параметры

Длина кода: 9, тип кода:Строка.

Длина наименования: 100.

Иерархический

Реквизиты: Вид действия - Создать и записать объект

  • Тип объекта. Строка. Тип создаваемого объекта.
  • Заполнить реквизиты значениями по умолчанию. Булево. Признак выполнения заполнения реквизитов создаваемого объекта, значениями по умолчанию.
  • Ввести объект на основании объекта-источника. Булево. Признак задействования механизма ввода на основании. Флаг доступен, если в конфигурации предусмотрена возможность ввода создаваемого объекта на основании объекта-источника события.
  • Провести документ. Булево. Признак выполнения проведения документа, в случае, если создается документ. 

Реквизиты: Вид действия - Создать напоминание

  • Дата начала. Дата. Дата оповещения как она заложена в напоминании без учета отсрочки напоминания пользователем и предварительного вывода напоминания.
  • Содержание. Строка. Сообщение содержащее суть напоминания.

Реквизиты: Вид действия - Создать запись в журнале регистрации

  • Регистрируемое событие. Строка. Произвольный текст, описывающий регистрируемое событие.
  • Уровень важности. Строка. Уровень важности регистрируемого события.
  • Объект метаданных. Объект метаданных. Объект метаданных, к которому относится событие.
  • Данные. Любая ссылка. Данные, с которыми связано событие.
  • Комментарий. Строка. Произвольный комментарий к записи в журнале.  

Реквизиты: Вид действия - Создать электронное письмо

  • Автор. «Справочник Пользователи». Реквизит является обязательным для заполнении. Пользователь, являющийся автором создаваемого письма.
  • Учетная запись. «Справочник Учетная запись электронной почты». Реквизит является обязательным для заполнения. Учетная запись, через которую будет отправлено письмо.
  • Тема. Строка. Реквизит является обязательным для заполнения. Тема письма.
  • Важность. Значение перечисления «Важность». Определяет важность создаваемого электронного письма.
  • Вариант записи. Значение перечисления «ВариантЗаписиЭлектронногоПисьма». Определяет действия над письмом после его создания. Письмо либо помещается в папку "Исходящие", либо производится попытка немедленной отправки.
  •  Формат текста. Значение перечисления «ФорматТекста». Определяет формат текста электронного письма: простой текст, либо HTML текст.
  •  Текст письма. Строка. Произвольный текст письма.
  • Комментарий. Строка. Произвольный комментарий к записи в журнале.  

Реквизиты: Вид действия - Создать сообщение SMS

  • Автор. «Справочник Пользователи». Реквизит является обязательным для заполнении. Пользователь, являющийся автором создаваемого сообщения.
  • Тема. Строка. Реквизит является обязательным для заполнения. Тема сообщения.
  • Важность. Значение перечисления «Важность». Определяет важность создаваемого сообщения.
  • Вариант записи. Определяет действия над сообщения после его создания. Письмо либо помечается как черновик, либо производится попытка немедленной отправки.
  •  Комментарий. Строка. Произвольный комментарий к сообщению.
  •  Текст сообщения. Строка. Произвольный текст сообщения.

Табличная часть «Правила заполнения»

Табличная часть содержит правила заполнения реквизитов создаваемого объекта.

  • Реквизит. Строка. Реквизит создаваемого объекта, значение которого необходимо установить.
  • Вид правила. Способ заполнения реквизита.
  • Правило. В зависимости от выбранного вида правила, может хранить точное значение реквизита, либо "ссылаться" на реквизит объекта-источника, либо содержать произвольный код на языке 1С.

Табличная часть «Получатели напоминания»

Табличная часть содержит правила вычисления получателей напоминания.

  • Вид правила. Способ вычисления получателя.
  • Значение. В зависимости от выбранного вида правила, может хранить ссылку на справочник «Пользователи» , либо "ссылаться" на реквизит объекта-источника, тип значения которого справочник «Пользователи», либо содержать произвольный код на языке 1С.

Табличная часть «Получатели письма»

Табличная часть содержит правила вычисления электронных адресов - получателей создаваемого электронного письма.

  • Вид правила. Способ вычисления электронного адреса.
  • Адрес электронной почты. В зависимости от выбранного вида правила, может содержать точное значение эл.адреса , либо содержать произвольный код на языке 1С вычисляющий эл. адрес.
  • Группа адреса. Значение перечисления "КодГруппыАдресаЭлектронногоПисьма".  Определяет поле, в которое будет подставлен вычисленный эл. адрес. (Кому, Копия, Скрытая копия).

Формы

  • Форма списка. Основная форма выбора. Используется для выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов.
  • Форма группы. Основная форма группы. Используется для заполнения обязательных реквизитов группы.
  • ВыборТипаОбъекта. Форма для выбора вида создаваемого объекта.  
  • РедактированиеКода. Форма для редактирования произвольного кода.  

Способ вызова

Работа со справочником производится только через форму элемента справочника «Значимые события».

Документ "Заявка на хранение шин"

Назначение

Документ «Заявка на хранение шин» предназначен для оформления предварительной заявки на хранение шин. На основании заявки документом «Приходный складской ордер» оформляется передача шин на хранение. Для возврата шин клиенту на основании заявки вводится «Расходный складской ордер». Документом «Реализация товаров» отражается факт оказания услуг по хранению шин. В случае необходимости на основании Реализации товаров выписывается счет-фактура.

Параметры

Длина номера: 10.

Периодичность номера: Год.

Тип номера: Строка.

Вводится на основании: «Заказ-наряд».

Объекты, вводимые на основании: «Приходный складской ордер», «Расходный складской ордер», «Переразмещение товаров», «Инвентаризация ордерного склада», «Реализация товаров».

Реквизиты

  • Склад. Ссылается на справочник «Склады (места хранения) компании». Склад компании, на котором предположительно будут хранится шины. В дальнейшем в документе «Расходный складской ордер» может быть выбран другой склад. Допускается выбор ордерного склада, использование розничного склада запрещено.
  • Ячейка. Ссылается на справочник «Ячейки хранения». Ячейка склада, в которой предположительно будут храниться шины. Заполняется, когда в правах выбрано хранение шин комплектами. В этом случае все шины в заявке располагаются в данной ячейке.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагенты и контакты». Клиент, который сдает шины на хранение.
  • Договор. Реквизит является обязательным для заполнения. Ссылается на справочник «Договоры взаиморасчетов». Договор, по которому ведутся взаиморасчеты с контрагентом.  
  • Автомобиль. Ссылается на справочник «Автомобили». Автомобиль клиента, на который установлены шины.
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ.
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ.
  • Менеджер. Ссылается на справочник «Сотрудники». Сотрудник, заключивший сделку с контрагентом.
  • Документ основание. Ссылается на документы «Заказ-наряд». Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов (по кнопке ).
  • Тип Цен. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы цен». Тип цен, в которых оказываются услуги по хранению шин.
  • Валюта. Реквизит является обязательным для заполнения. Ссылается на справочник «Валюты». Валюта документа.
  • Курс. Реквизит является обязательным для заполнения. Тип значения: «Число». Курс валюты документа, на дату документа.
  • Курс валюты упр. учета. Тип значения: «Число». Курс валюты управленческого учета. (Возможность редактирования задается правом  "Редактирование курса управленческой валюты в документах").
  • Карточка. Ссылается на справочник «Карточки». Карточка покупателя, по которой оказываются услуги по хранению шин.
  • Скидка/наценка. Ссылается на справочник «Типы скидок и наценок». Скидка/наценка документа.
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.
  • Сумма. Тип значения: «Число». Сумма документа.
  • Дата начала. Реквизит является обязательным для заполнения. Тип значения: «Дата». Дата начала хранения шин.
  • Дата окончания. Реквизит является обязательным для заполнения. Тип значения: «Дата». Дата окончания хранения шин.

Табличная часть «Услуги»

  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Услуги, оказываемые в рамках заявки по хранению шин.
  • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество оказываемых услуг.
  • Единица измерения. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Единицы измерения». Единица измерения услуги.
  • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент пересчета по отношению к базовой единице услуги.
  • Цена. Тип значения: «Число». Цена услуги по хранению.
  • Сумма. Тип значения: «Число». Сумма всегда без скидки. НДС входит в сумму если это указано в типе цен. Сумма всегда равна Цена*Количество.
  • Процент скидки. Тип значения: «Число». Процент скидки на услугу.
  • Скидка. Тип значения: «Число». Сумма скидки по строке документа.
  • Всего. Тип значения: «Число». Итоговая сумма по строке документа со всеми налогами.
  • Процент НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по услуге.
  • НДС. Тип значения: «Число». Сумма НДС по услуге.
  • Характеристика номенклатуры. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика услуги.    

Табличная часть «Шины»

  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Шина, принимаемая на хранение.
  • Диск. Тип значения: «ПеречислениеСсылка.ВидыАвтомобильныхДисков». Вид диска, установленного вместе с шиной. 
  • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество шин.
  • Единица измерения. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Единицы измерения». Единица измерения шины.
  • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент пересчета по отношению к базовой единице.
  • Расположение шины. Реквизит является обязательным для заполнения. Тип значения: «ПеречислениеСсылка.РасположениеШин». Расположение шины на автомобиле.
  • Характеристика номенклатуры. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика шины в зависимости от выбранного способа учета характеристик для данной номенклатуры (вида номенклатуры).
  • Примечание. Тип значения: «Строка». Описание шины, сдаваемой на хранение. В примечание могут быть добавлены значения из справочника «Дефекты» с помощью кнопки выбора. Выбранные в справочнике значения добавляются в примечание в виде строки.
  • Ячейка. Ссылается на справочник «Ячейки хранения». Ячейка хранения для номенклатуры. Ячейка хранения, в которой предполагается хранение шины. Заполняется, когда в правах запрещено хранение шин комплектами. В этом случае все шины в заявке могут располагаться в различных ячейках.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора документов данного типа.
  • Форма документа. Основная форма документа. Используется для заполнения обязательных реквизитов.

Печатные формы

  • Акт приема-передачи. Акт приема-передачи шин на сезонное хранение. Содержит перечень шин и их характеристики. Является частью договора на хранение шин.
  • Бирки. Бирки для хранения. Печатаются на каждую шину индивидуально, используются для маркировки шин на складе. 
  • Договор хранения. Договор сезонного хранения шин.
  • Договор хранения (шаблон). Договор сезонного хранения шин в виде шаблона печатной формы для Microsoft Office и Open Office.
  • Заявка на хранение шин. Бланк заявки для ручного заполнения мастером автосервиса.

Поиск характеристик шин
Для поиска можно указать в строке шин следующие значения:
  • Число. Для поиска по радиусу, ширине или высот протектора;
  • "з", "зима", "w", "winter". Для поиска зимних шин;
  • "ш", "шины", "шипованная". для поиска шин с шипами.

Способ вызова

Вызывается из меню "Документы --> Складские документы--> Заявка на хранение шин".

Раскраска строк табличной части "Шины" в форме списка

 

Цвет текста строк табличной части "Шины" зависит от состояния документа по следующим правилам:

Строка

Состояние Расшифровка состояния
новая заявка Документ находится в состоянии "новая заявка", когда шины на основании этой заявки еще не приходовались на склад и не выдавались клиенту.
шины приняты Документ находится в состоянии "шины приняты", когда на основании этой заявки был введен документ «Приходный складской ордер» и шины, оприходованным этим ордером, числятся на складе.
шины выданы Документ находится в состоянии "шины выданы", когда все переданные на хранение по этой заявке шины были выданы клиенту.

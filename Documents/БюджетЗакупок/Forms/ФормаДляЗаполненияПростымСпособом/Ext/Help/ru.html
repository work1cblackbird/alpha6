Произвольная форма "Наивное прогнозирование"

Назначение

Произвольная форма документа «Бюджет закупок», предназначена для заполнения документа данными, используя метод наивного прогнозирования. Наивное прогнозирование заключается в получении простого арифметического среднего прогнозируемого ресурса (количество или сумма упр.) от фактических или запланированных данных предыдущих периодов.

Способ вызова

Форма вызывается из меню командной панели табличной части документа «Бюджет закупок» : "Наивное прогнозирование" или из командной панели другой произвольной формы документа "Заполнение по документу основанию".

Принцип работы с формой

Для начала работы с формой необходимо определиться - арифметические средние от каких данных необходимо получить.  Предусмотрены четыре типа данных для анализа:

  1. По фактическим данным закупок. Анализируется приход по регистру  «Партии товар компании».
  2. По фактическим данным продаж. Анализируется оборот по регистру  «Продажи».
  3. По ранее планируемым данным закупок. Анализируется оборот по регистру  «Бюджет закупок».
  4. По ранее планируемым данным продаж. Анализируется оборот по регистру  «Бюджет продаж».
Выбор нужного типа осуществляется при помощи реквизита "Тип анализа".

Выбрать количество анализируемых периодов.  Данный параметр фактически определяет количество значений, от которых будет получено среднее арифметическое. Периоды отсчитываются назад от даты планирования документа «Бюджет закупок», из которого была вызвана данная форма. Вид периода (его размерность: месяц, квартал, год) определяется периодичностью выбранного в документе сценария планирования.  

Выбрать тип учета остатков. Определяет, каким образом будут использоваться ограничения складских остатков номенклатуры на спрогнозированные данные. Возможны 4 варианта:

  1. Без учета ограничений - ограничения на спрогнозированные данные не накладываются.
  2. Количество не меньше минимального остатка - спрогнозированные данные будут ограничены "снизу" значением минимального остатка.
  3. Количество, увеличенное на минимальный остаток - к прогнозному значению количества будет добавлено значение минимального остатка.
  4. Количество не меньше минимального и не больше максимального - прогнозное значение будет ограничено "сверху" и "снизу".

Значения ограничений берутся из периодического регистра сведений «Дополнительные реквизиты номенклатуры» на дату документа.

Далее необходимо установить коэффициенты прогнозирования: Коэффициент роста и коэффициент сезонности. Эти коэффициенты определяются экспертным путем на основании предположения о существенном увеличении\уменьшении закупок в планируемом периоде. Выражаются в натуральной величине (не в процентах).

Показатель планирования - определяет ресурс, который будет запланирован (количество или сумма упр.). Т.е. обороты какого ресурса будут рассчитаны. В дальнейшем через значение цены (соответствующей выбранному типу цен) будет найден второй ресурс.  В случае категорийного бюджета показатель планирования не доступен, т.к. жестко планируется "сумма упр.".

Способ округления  - определяет режим округления количества для весового товара. Невесовой товар округляется в любом случае.

Данное прогнозирование используется как для номенклатурного, так и для категорийного бюджетов закупок. Для категорийного бюджета, в случае если выбран тип анализа "по фактическим данным", после получение результата, данные сворачиваются по категориям номенклатуры (по типу номенклатуры). В случае, если выбран тип анализа "по ранее планируемым данным", результат сразу получается по категориям (различие связано со структурой регистров)

Реквизиты

  • Количество прошлых периодов. Число. Количество анализируемых периодов.
  • Коэффициент продаж. Число. Экспертная оценка роста объема закупок в прогнозируемом периоде.
  • Коэффициент сезонности. Число. Экспертная оценка сезонной составляющей.
  • Показатель планирования. Булево. Вид прогнозируемого ресурса ("количество" или "сумма упр.")
  • Способ округления. Число. Определяет способ округления ресурса "количество".
  • Сценарий планирования. Справочник  «Сценарий планирования». Соответствует сценарию планирования документа  «Бюджет закупок», из которого была вызвана форма. В форме не редактируется. 
  • Способ учета остатков.  Перечисление «Способы учета остатков».
  • Тип анализа. Перечисление  «Типы анализа». Определяет вид анализируемых данных.
  • Тип номенклатуры. Список значений. Содержит список типов номенклатуры. Используется для наложения ограничений на прогнозируемые данные.
  • Форма мастера. Произвольный. Используется при вызове данной формы из формы заполнения по документу основанию. Содержит ссылку на эту форму.

Произвольная форма "Прогнозирование математическими методами"

Назначение

Произвольная форма документа «Бюджет продаж», предназначена для заполнения документа данными, используя некоторые методы математического прогнозирования. Суть работы этого заполнения заключается в следующем: получается временной ряд для каждой номенклатуры/категории номенклатуры. Временной ряд - в данном случае, значения оборотов ресурса: количества или суммы упр., полученные за определенные периоды, допустим по месяцам. На основании данных временных рядов строятся математические модели. Далее, согласно математическим моделям находятся прогнозные значение ресурса для каждой номенклатуры\категории номенклатуры.

Способы вызова

Форма вызывается из меню командной панели табличной части документа «Бюджет продаж» : "Прогнозирование мат. методами".

Принцип работы с формой

Выбирается количество анализируемых периодов - "Количество периодов для анализа".  Периоды отсчитываются назад от даты планирования документа (не путать с датой документа!) «Бюджет продаж», из которого была вызвана данная форма. Вид периода (его размерность: месяц, квартал, год) определяется периодичностью выбранного в документе сценария планирования.  

Показатель планирования - определяет ресурс, который будет запланирован (количество или сумма упр.). Т.е. обороты какого ресурса будут рассчитаны. В дальнейшем, через значение цены, соответствующей выбранному типу цен, будет найден второй ресурс. В случае категорийного бюджета показатель планирования не доступен, т.е. жестко планируется сумма упр..

Способ округления  - определяет режим округления количества для весового товара. Невесовой товар округляется в любом случае.

Далее необходимо выбрать нужную математическую модель и заполнить ее параметры. Состав и назначение параметров различно для каждой модели прогнозирования, и выбираются экспертным путем. Некоторые модели имеют ограничение на минимальное значение количества анализируемых периодов. Помимо этого, для некоторых моделей можно задать анализ сезонной составляющей временного ряда, однако для этого должно быть выбрано достаточное количество периодов. Сезонная составляющая характерна для продаж, объемы которых сильно зависят от времени года.

Описание предлагаемых математических моделей

  • Прогноз средним скользящим. Представляет собой простое среднее арифметическое от значений временного ряда. Параметров нет. Недостатки: плохо учитывает трендовую составляющую, не учитывает сезонные колебаний, то есть сезонную компоненту.
  • Прогноз сглаживания Хольта-Винтерса. Фактически представляет собой две модели: модель Хольта и модель Винтерса. Выбор того или иного метода происходит автоматически, в зависимости от возможности анализа сезонной компоненты.
    • Метод Хольта. Является развитием метода экспоненциального сглаживания. Позволяет учитывать локальный линейный тренд, присутствующий во временных рядах. Параметры: постоянная сглаживания для данных - число, от 0 до 1, оценивает текущий уровень временного ряда; Постоянная сглаживания тренда - число, от 0 до 1, оценивает наклон временного ряда. Недостаток: не учитывает сезонную составляющую временного ряда.
    • Метод Винтерса. Является развитием метода Хольта, позволяет оценивать сезонную составляющую временного ряда. Параметр: постоянная сглаживания для данных - число, от 0 до 1, оценивает текущий уровень временного ряда; Постоянная сглаживания тренда - число, от 0 до 1, оценивает наклон временного ряда; постоянная сглаживания сезонности - число, от 0 до 1, оценка сезонной составляющей во временном ряду.
  • Прогноз линейным трендом. Получает уравнение прямой, наиболее точно описывающее исследуемый ряд. Является частным случаем модели "Прогнозирование полиномиальным трендом". Прогнозное значение получается подстановкой в уравнение номера прогнозируемого периода. Параметров не имеет.
  • Прогноз экспоненциальным трендом. Получает уравнение экспоненциальной кривой, наиболее точно описывающей исследуемый ряд.  Прогнозное значение получается подстановкой в уравнение номера прогнозируемого периода. Параметров не имеет. Ограничение на значения временного ряда: не должно присутствовать нулевых значений.
  • Прогноз полиномиальным трендом. Получает уравнение полиномиальной кривой, наиболее точно описывающей исследуемый ряд. Прогнозное значение получается подстановкой в уравнение номера прогнозируемого периода. Параметры: степень полинома - число (от 2 до 6).  Ограничение: выбранное количество периодов для анализа должно быть не меньше квадрата выбранной степени полинома.

 

Реквизиты

  • Количество прошлых периодов. Число. Количество анализируемых периодов.
  • Модель прогнозирования.Перечисление «Модели прогнозирования». Математическая модель, применяемая при прогнозировании.
  • Параметр1. Число. Параметр математической модели.
  • Параметр2. Число. Параметр математической модели.
  • Параметр3. Число. Параметр математической модели.
  • Показатель планирования. Булево. Вид прогнозируемого ресурса (количество, либо сумма упр.)
  • Расчет сезонности. Булево. Признак учета сезонной составляющей временного ряда при построении модели.
  • Способ округления. Число. Способ округления количества.

 

Дерево значений "Номенклатура" ("Категории номенклатуры")

Предназначено для отображения фактических и спрогнозированных значений, а также для оценки адекватности построенной математической модели, используя значения средней квадратической ошибки.

  • Номенклатура / категория номенклатуры - на первом уровне отображает номенклатуру / категорию номенклатуры, на втором уровне - временной ряд для номенклатуры.
  • Количество / Сумма (упр) (факт.знач) - на втором уровне отображает значения временного ряда (т.е. фактические значения количества или суммы упр)
  • Количество / Сумма (упр) (знач. модели) - на втором уровне отображает значения построенной модели
  • Прогноз - на первом уровне отображает прогнозное значение ресурса для текущей номенклатуры / категории номенклатуры.
  • Ср. квадратичная ошибка (MSE) - отображает средне квадратичную ошибку построенной математической модели в сравнении с рядом фактических значений. Позволяет судить об адекватности модели, сравнивать модели между собой в точности.

 

Дополнительная информация

Предусмотрена возможность в графическом сравнении фактического ряда и спрогнозированного ряда. Кнопка "График".

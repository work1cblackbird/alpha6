Произвольная форма "Заполнение по документу основанию"

Назначение

Произвольная форма документа «Бюджет продаж», предназначена для заполнения документа данными, на основании другого документа «Бюджет продаж». Позволяет вводить категорийный бюджет на основании категорийного бюджета, номенклатурный бюджет на основании номенклатурного бюджета, а также номенклатурный бюджет на основании категорийного бюджета. В последнем случае предлагается выполнить распределение сумм категорий документа основания по суммам номенклатурных позиций,  которые в свою очередь заполняются методами наивного прогнозирования, либо вручную.

Способы вызова

Форма вызывается из меню командной панели табличной части документа «Бюджет продаж»: "Заполнение по документу основанию", а также в случае ввода номенклатурного бюджет на основании категорийного бюджета штатными средствами платформы 1С.

Принцип работы с формой

В случае если заполняемый документ имеет вид бюджета "Бюджет продаж по категориям", то вся работа с формой сводится к выбору документа основания. В этом случае в качестве документа основания может выступать только документ «Бюджет продаж» с аналитикой по категориям, причем периодичности сценариев планирования документов должны совпадать. Также должно выполняться условие, что период планирования документа основания должен быть не больше периода планирования заполняемого документа. Далее таблица формы заполнится табличной частью документа основания и будет недоступна для редактирования. При нажатии "ОК" данные таблицы скопируются в табличную часть заполняемого документа.

Если происходит заполнением бюджета продаж с аналитикой по номенклатуре ("Бюджет продаж по номенклатуре"), то в качестве основания может выступать документ «Бюджет продаж» с любой аналитикой , при этом накладываются те же ограничения на периодичность сценариев планирования и дату планирования документа основания. Если в качестве документа основания выбран «Бюджет продаж» с аналитикой по номенклатуре, т.е. бюджеты совпадают, то дальнейшая работа с формой будет аналогично предыдущему случаю. Если же выбран «Бюджет продаж» с аналитикой по категориям номенклатуры, то дальнейшая работа с формой будет иной. В этом случае таблица на форме будет отображаться в виде дерева значений. На первом уровне будут находиться данные по категориям номенклатуры, которые  заполнятся по табличной части документа основания. Второй уровень будет содержать данные по номенклатуре, которые заполняются вручную, либо при помощи заполнения наивным прогнозированием - кнопка "Заполнить" на командной панели. Номенклатура добавляется строго в свою "ветку" с учетом категории, к которой она принадлежит. В итоге мы имеем двухуровневую таблицу. В колонке "Сумма базовая" отображается сумма категории, в колонке "Сумма (упр)" содержатся суммы номенклатур. Далее необходимо распределить суммы категорий по суммам номенклатур, то есть "подогнать" суммы номенклатур таким образом, чтобы итоговые значения "Сумма упр" по каждой категории номенклатур максимально соответствовал значению базовой суммы. Распределение производится по весовым коэффициентам, которые могут рассчитываться тремя способами: по количеству номенклатуры, по сумме номенклатуры, по рентабельности номенклатуры  (реквизит "Метод распределения сумм продаж категорий"). Также можно выбрать показатель, который будет изменяться в ходе "подгонки" сумм: Количество или Цена номенклатуры.

Реквизиты

  • Документ основание. Ссылка на документ «Бюджет продаж».
  • Количество прошлых периодов. Тип значения: «Число». Количество анализируемых периодов. Используется при вызове формы наивного прогнозирования.
  • Коэффициент роста продаж. Тип значения: «Число». Экспертная оценка роста объема продаж в прогнозируемом периоде.  Используется при вызове формы наивного прогнозирования.
  • Коэффициент сезонности. Тип значения: «Число». Экспертная оценка сезонной составляющей.  Используется при вызове формы наивного прогнозирования.
  • Показатель планирования. Перечисление  «Типы анализа». Вид прогнозируемого ресурса (количество или сумма упр.)  Используется при вызове формы наивного прогнозирования.
  • Способ округления. Тип значения: «Число». Определяет способ округления ресурса "количество"
  • Сценарий планирования. Справочник  «Сценарий планирования». Соответствует сценарию планирования документа  «Бюджет продаж», из которого была вызвана форма. В форме не редактируется. 
  • Тип анализа. Перечисление  «Типы анализа». Определяет вид анализируемых данных: фактические данные продаж, либо ранее запланированные данные продаж.  Используется при вызове формы наивного прогнозирования.
  • Метод распределения категория. Перечисление  «Методы распределения категорий» . Способ расчета весовых коэффициентов номенклатуры.
  • Параметр управления распределением. Тип значения: «Булево». Определяет ресурс, который будет изменяться в ходе распределения (количество, либо цена)

 

Дополнительная информация

В случае если происходит распределение сумм категорий, и в качестве изменяемого показателя выбрано "количество", то точной "подгонки" в общем случае не получится для невесового товара, т.к. невесовой товар может принимать только дискретные значения.

Документ "Бюджет Продаж автомобилей"

Назначение

Документ "Бюджет продаж автомобилей" предназначен для планирования продаж моделей или вариантов комплектации автомобилей за выбранный период времени, в определенном подразделении и для выбранного сценария планирования, который определяет периодичность. Документ позволяет запланировать данные, как по моделям, так и по вариантам комплектации.  Вид планируемой аналитики определяется хоз. операцией, т.е. выбранным видом бюджета. В зависимости от выбранного вида бюджета табличная часть документа именуется либо "Варианты комплектации" либо "Модели".  Запланированные данные продаж относятся к выбранному в документе периоду планирования, который соответствует периодичности сценария планирования (поддерживаются следующие периоды планирования: месяц, квартал, год). Планируемый период не может быть меньше даты документа.

Возможность ввода "Бюджета продаж автомобилей" для того или иного подразделения определяется настройкой регистра сведений «Параметры планирования», и жестко контролируется.

Суммы продаж планируются в валютах управленческого и регламентированного учета, пересчет из одной валюты в другую осуществляет по курсу на дату документа.

Документ позволяет производить ручное заполнение, то есть непосредственное добавление элементов в табличную часть и автоматическое заполнение, на основании фактических или запланированных данных продаж предыдущих периодов, с использованием методов математического прогнозирования или на основании документов  «Бюджет продаж автомобилей». Автоматическое заполнение реализовано в виде набора дополнительных форм документа.

При проведении документ формирует хозяйственные операции:

  • Бюджет продаж по вариантам комплектации. Формируются движения по регистру накопления: «Бюджет продаж автомобилей», в котором производится накопление запланированных данных продаж.  Измерение "Вариант комплектации" и "Модель" заполняется позициями  из табличной части Автомобили. 
  • Бюджет продаж по моделям. Формируются движения по регистру накопления: «Бюджет продаж автомобилей», в котором производится накопление запланированных данных продаж. Измерение "Модель" заполняется позициями табличной части Автомобили, а измерение "Вариант комплектации" остается незаполненным.

Параметры

Длина номера: 10

Периодичность номера: Год

Тип номера: Строка

Вводится на основании:«Бюджет продаж автомобилей».

Объекты, вводимые на основании: «Бюджет продаж автомобилей», «Корректировка»

Реквизиты

  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ.
  • Дата планирования. Реквизит является обязательным для заполнения. Тип значения: «Дата». Начальная дата периода, на который происходит планирование. Зависит от реквизита "Периодичность" выбранного сценария планирования. 
  • Документ основание.  Тип значения: «Бюджет продаж автомобилей». Содержит документ-основание, в случае, если применялось автоматическое заполнение по документу-основанию.
  • Смещение планирования. Тип значения: «Число». Количество отступаемых периодов назад от планируемого периода при использовании автоматического заполнения. Размерность периода зависит от реквизита "Периодичность" выбранного сценария планирования.
  • Количество периодов. Тип значения: «Число». Количество анализируемых периодов при использовании автоматического заполнения. Размерность периода зависит от реквизита "Периодичность" выбранного сценария планирования. Периоды отсчитываются назад от смещения периодов.
  • Количество сезонов. Тип значения: «Число». Количество анализируемых сезонов при использовании автоматического заполнения. Выбранный период прогнозирования будет рассчитываться за несколько сезонов (лет).
  • Не учитывать периоды без данных. Тип значения: «Булево». Флаг активизирует возможность пропуска периодов в которых не было продаж.
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу. 
  • Коэффициент роста. Тип значения: «Число». Используется при автоматическом заполнении документа методами наивного прогнозирования  и характеризует рост либо снижение объема продаж в планируемом периоде. При возрастании объема продаж коэффициент больше 1, при снижении объема продаж коэффициент меньше 1.
  • Коэффициент сезонности.  Тип значения: «Число». Используется при автоматическом заполнении документа методами наивного прогнозирования и характеризует рост или снижение объема продаж в планируемом периоде. Данный коэффициент должен характеризовать тенденцию роста или снижения продаж в зависимости от планируемого сезона. Вместе с коэффициентом роста продаж образуется мультипликативная модель прогнозирования, то есть коэффициенты перемножаются. 
  • Курс документа. Тип значения: «Число». Равен отношению курсов валюты типа цен и валюты управленческого учета, на дату документа. Ручное редактирование реквизита необходимо, если предполагается изменение курсов валют в прогнозируемом периоде. Редактирование выполняется в дополнительной форме выбора типа цен. Доступно, только, если у выбранного типа цен не установлен флаг  "В валюте автомобиля", иначе коэффициент пересчета валют рассчитывается для каждого автомобиля в отдельности и не редактируется.
  • Метод распределения категорий. Ссылка на перечисление «Методы распределения категорий». Используется в случае заполнения вариантного бюджета продаж на основании модельного при распределении суммы моделей по суммам позиций вариантов комплектации.
  • Модель прогнозирования.  Ссылка на перечисление «Модели прогнозирования». Используется при заполнении методами математического прогнозирования. Определяет используемую математическую модель для обработки временного ряда.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ.
  • Параметр1. Тип значения: «Число». Используется при автоматическом заполнении методами математического прогнозирования. Определяет степень полинома при прогнозировании полиномиальным трендом или постоянную сглаживающую для данных при прогнозировании методом Хольта-Винтерса. Для остальных математических моделей неопределено.
  • Параметр2. Тип значения: «Число». Используется при автоматическом заполнении методами математического прогнозирования. Определяет постоянную сглаживающую тренда при прогнозировании методами Хольта-Винтерса. Для остальных математических моделей неопределено.
  • Параметр3. Тип значения: «Число». Используется при автоматическом заполнении методами математического прогнозирования. Определяет постоянную сглаживающую сезонности при прогнозировании методами Хольта-Винтерса. Для остальных математических моделей неопределено.
  • Параметр управления распределением. Тип значения: «Булево». Используется при заполнении вариантного бюджета на основании модельного бюджета. Определяет реквизит табличной части (количество или цена), который будет изменяться при распределении суммы модели по суммам вариантов конфигурации.
  • Планирование себестоимости. Тип значения: «Булево». Признак планирования себестоимости в документе. Устанавливается при при усеченной схеме операционных бюджетов, когда не составляется отдельный бюджет закупок автомобилей.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ. 
  • Показатель планирования. Тип значения: «Булево». Используется при автоматическом заполнении. Определяет планируемые показатель: количество либо сумма.   
  • Расчет сезонности. Тип значения: «Булево». Используется при автоматическом заполнении методами математического прогнозирования. Определяет признак учета сезонной составляющей временного ряда для некоторых математических моделей прогнозирования, если выбранное количество периодов достаточно для анализа.
  • Себестоимость упр. итого. Тип значения: «Число». Итоговое значение себестоимости в валюте управленческого учета, в случае, если планируется себестоимость. 
  • Способ последнего заполнения. Тип значения: «Число». Определяет способ последнего заполнения, необходимо для формирования строки с информацией о последнем используемом автоматическом способе заполнения.
  • Сумма документа. Тип значения: «Число». Итоговое значение суммы планирования в валюте управленческого учета. 
  • Сценарий планирования.  Реквизит является обязательным для заполнения. Ссылается на справочник «Сценарии планирования». Определяет периодичность планирования.
  • Тип анализа.  Ссылка на перечисление «Типы анализа». Используется при автоматическом заполнении. Определяет тип анализируемых данных. 
  • Тип цен. Ссылается на справочник «Типы цен». Тип цен.
  • Хозяйственная операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция. Определяет бюджет планирования (Бюджет продаж по вариантам комплектации, либо бюджет продаж по моделям) 

    Табличная часть «Автомобили»

    Основная табличная часть документа, предназначенная для хранения запланированных данных.
    • Модель. Ссылается на справочник «Модели». Прогнозируемая аналитика.
    • Вариант комплектации. Ссылается на справочник «Варианты комплектации». Прогнозируемая аналитика.
    • Цена. Тип значения: «Число». Цена автомобиля на дату документа, соответствующая выбранному типу цен.
    • Количество.Тип значения: «Число». Запланированное количество автомобилей.
    • Сумма упр. Тип значения: «Число». Запланированная сумма продаж автомобилей в валюте управленческого учета.  
    • Себестоимость упр. Тип значения: «Число». Запланированная сумма себестоимости модели\варианта комплектации в валюте управленческого учета. 
    • Сумма НДС. Тип значения: «Число». Запланированная сумма НДС продаж в валюте регламентированного учета.
    • Нормативная цена. Тип значения: «Число». Нормативная цена номенклатуры. Рассчитывается из запланированной суммы себестоимости. 

    Формы

    • Форма списка. Основная форма списка. Используется для отображения списка или выбора документов данного типа.
    • Форма документа. Основная форма документа. Используется для заполнения обязательных реквизитов.
    • Форма списка для выбора. Основная форма выбора.
    • Форма для заполнения простым способом. Произвольная форма. Предназначена для заполнения документа данными, используя метод наивного прогнозирования. Наивное прогнозирование заключается в получении простого арифметического среднего прогнозируемого ресурса (количество или сумма упр.) от фактических или запланированных данных предыдущих периодов. 
    • форма для заполнения математическими методами. Произвольная форма. Предназначена для заполнения документа данными, используя некоторые методы математического прогнозирования. Суть работы заключается в следующем: получается временной ряд для каждой модели/варианта комплектации (Временной ряд - в данном случае, значения оборотов ресурса (количества или суммы упр) полученные по периодам). На основании данных временных рядов строятся математические модели. Далее, согласно математическим моделям, находятся прогнозные значение ресурса для каждой модели\варианта комплектации.
    • Форма для заполнения по документу основанию. Произвольная форма. Предназначена для заполнения документа данными на основании документов «Бюджет продаж автомобилей». Позволяет производить распределение сумм моделей по суммам позиций вариантов комплектации, то есть от общего к частному. 
    • Форма для выбора типа цен. Произвольная форма. Используется для выбора типа цены, и коэффициента пересчета из валюты типа цены в валюту управленческого учета.
    • Результат. Произвольная форма. Используется для графического отображения фактического и смоделированного временных рядов при прогнозировании математическими методами.

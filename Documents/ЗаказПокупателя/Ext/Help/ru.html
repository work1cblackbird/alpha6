Документ "Заказ покупателя"

Назначение

Документ Заказ покупателя служит для отражения в учете заказов покупателей. В табличной части документа не могут фигурировать услуги, работы и т.п. Относится к подсистеме Заказы раздела Продажи.

При проведении документ формирует хозяйственные операции:

  • Заказ покупателя. Хозяйственная операция приходует заказы покупателя по списку заказанных товаров на регистр Заказы покупателей.
  • Заказ и резервирование покупателя. Хозяйственная операция аналогична операции Заказ покупателя, за тем исключением, что кроме приходования заказов, выполняется резервирование заказанных товаров по складу на регистре Остатки товаров компании.
  • Резервирование покупателя.  Хозяйственная операция выполняется резервирование заказанных товаров по складу на регистре Остатки товаров компании.

Реквизиты

  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник Контрагенты и контакты. Покупатель, заказывающий товар.
  • Договор. Реквизит является обязательным для заполнения. Ссылается на справочник Договоры взаиморасчетов. Договор, по которому ведутся взаиморасчеты с контрагентом.
  • Тип заказа. Один из возможных типов заказов. Выбирается из справочника Типы заказов.
  • Предоплата %, Сумма предоплаты. Процент и сумма предоплаты по заказу. В персональных настройках пользователя может быть задано ограничение на ввод минимального процента предоплаты по заказу покупателя, которое устанавливается по умолчанию при создании нового документа.
  • Резервировать на. Склад, на котором производится резервирование заказанного товара.
  • Основание. Ссылается на документы Счет на оплату, Поступление товаров. Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу-основанию. С помощью этого реквизита затем строится дерево подчиненных документов (по кнопке ).
  • Макс. срок поставки. Максимальный срок поставки поставки (значение по умолчанию берется из настроек выбранного подразделения). Справа от этого поля показывается надпись, детализирующая срок поставки.
  • Срок снятия резерва. Срок, после которого резерв товара может быть снят. Для хозяйственной операции Заказ покупателя это поле недоступно.
  • По договору долг контрагенту составляет. Гиперссылка отражает текущее состояние расчетов с контрагентом по договору. Производится переход к отчету Взаиморасчеты с контрагентами, сформированный для указанного контрагента.

Если документ Заказ покупателя вводится на основании документа Поступление товаров, либо без основания, в случае если для пользователя установлено право Основной тип цен продажи, то в документе по умолчанию будет установлен основной тип цен продажи.

Если для пользователя установлена настройка Минимальный процент предоплаты, то в документе по умолчанию будет установлен указанный минимальный процент предоплаты для данного пользователя.

Если для подразделения компании установлена настройка Срок поставки покупателю, то в документе будет установлен срок поставки покупателю по умолчанию.

Установленное для пользователя право Разрешить отгрузку при наличии задолженности по договору позволяет проводить документ Заказ покупателя при наличии задолженности по указанному договору взаиморасчетов на момент проведения документа.

Табличная часть «Товары»

Добавочные табличные графы:

  • Количество. Реквизит является обязательным для заполнения. Показывает количество заказанного товара. Для операции Резервирование покупателя этой графы в таблице нет, так как считается, что весь заказанный товар резервируется на указанном складе. Количество заказываемого товара не может быть меньше резервируемого количества.
  • Резерв. Показывает количество товара, резервируемого на складе. Для операции Заказ покупателя этой графы в таблице нет, так как резервирование не производится. Такое резервирование можно осуществить позже, при помощи документа Резервирование покупателя. Количество
    резервируемого товара не может быть больше заказываемого количества.

Кнопка Состояние может быть нажата или отжата. Если кнопка отжата, то вся таблица показывается черным шрифтом на белом фоне. Однако если кнопка нажата, то строки таблицы, соответствующие заказанным товарам, могут иметь различные цвета, показывающие состояние товара. Цвета имеют следующий смысл.

  • Серый. Товар полностью отгружен заказчику.
  • Синий. Заказанный товар не отгружен, но имеется на складе.
  • Зеленый. Заказанный товар не отгружен, но имеется на складе и зарезервирован для данного заказчика.
  • Желтый. Товара нет, но он распределен по заказам поставщикам.
  • Красный. Товара нет и он не распределен по заказам поставщикам (полностью или частично).

Ввод на основании

Вводится на основании документов:

  • Счет на оплату,
  • Поступление товаров.

Объекты, вводимые на основании документа Заказ покупателя:

  • Заказ внутренний,
  • Корректировка заказа покупателя,
  • Приходный кассовый ордер,
  • Реализация товаров,
  • Резервирование заказов покупателя,
  • Корректировка движений,
  • Снятие резервов заказов покупателя,
  • Счет на оплату,
  • Заказ поставщику,
  • Чек на оплату,
  • Жалоба клиента,
  • Замена в заказе покупателя,
  • Перемещение резервов,
  • Перемещение товаров,
  • План поступления ДС,
  • Распределение заказа покупателя,
  • Расходный ордер на склад,
  • Снятие распределения заказов покупателя.

Печатные формы

  • Заказ покупателя. Простая печатная форма. Выводит список заказанных товаров. 

Журналы, отображающие документ

  • Операции с покупателями,
  • Заказы и резервы.

Документ "Заявка на ремонт"

Назначение

Документ Заявка на ремонт служит для планирования загрузки ремонтных зон и предварительного расчета стоимости ремонта. Относится к подсистеме Автосервис раздела Автосервис.

С помощью этого документа можно, например, организовать очередь обращений клиентов, если мощностей автосервиса не хватает, чтобы обслуживать всех клиентов в момент обращения, или организовать учет согласованного с клиентом времени выполнения работ. В дальнейшем внесенные в базу данных заявки на ремонт можно проанализировать с помощью отчета Отчет по записи на ремонт.

Список хозяйственных операций:

  • Заявка на ремонт,
  • План ремонта.

Вкладка «Основное»

На вкладке Основное приводятся данные:

  • о клиенте-заказчике ремонта,
  • ремонтируемом автомобиле,
  • ремонте, который требуется произвести,
  • предполагаемом периоде ремонта,
  • маркетинговой программе, в рамках которой планируется ремонт.

Табличная часть «Автоработы»

В левой части вкладки Автоработы указывается причина обращения в автосервис. Правая табличная часть содержит список авторабот, которые соответствуют данной заявке на ремонт.

Табличная часть этой вкладки может быть заполнена:

  • вручную,
  • с помощью кнопок Добавить или Подбор,

  • автоматически при помощи пунктов меню кнопки Заполнение, расположенной над данной таблицей:

    • Связанные работы – таблица заполняется информацией о связанных работах, хранящихся в регистре сведений Связанные работы;

    • Работы по виду ремонта – таблица заполняется списком работ по умолчанию, указанных для данного вида ремонта.

Реквизиты данной вкладки:

  • Авторабота. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник Автоработы. Работа для выполнения заказа.
  • Количество. Реквизит является обязательным для заполнения. Количество работ.
  • Нормочас. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник Нормочасы. Нормочас работы.
  • Коэффициент. Реквизит является обязательным для заполнения. Цена нормочаса.
  • Цена. Цена работы в валюте документа.
  • Сумма. Сумма всегда без скидки. НДС входит в сумму, если это указано в типе цен. Сумма всегда равна Цена*Количество*Коэффициент.
  • Ставка НДС. Ссылается на справочник Ставки НДС. Ставка НДС по работе.
  • Сумма НДС. Сумма НДС по работе.
  • Процент скидки. Процент скидки на работу.
  • Сумма скидки. Сумма скидки на работу.
  • Всего. Итоговая сумма по строке документа со всеми налогами.
  • Способ расчета стоимости работы. Указывает каким образом будет произведен расчет суммы автоработы. Если установлен признак "Фиксированной суммой", то сумма автоработы рассчитывается по формуле Количество * Цена. Иначе - Количество * Норма времени * Цена
  • Исполнители. Исполнители работ.

При выборе на этой вкладке исполнителей ремонта открывается форма Подбор исполнителей, содержащая следующие реквизиты:

  • Исполнитель. Сотрудник, участвующий в выполнении автоработы. Выбирается из справочника Сотрудники. Имеют право на проведение ремонта те сотрудники, у которых в карточках сотрудника установлен флажок Исполнитель.
  • Цех. Место выполнения указанной автоработы данным сотрудником. Выбирается из справочника Цеха.
  • Процент. Процент участия данного сотрудника в выполнении соответствующей автоработы. Эта информация будет учтена при расчете заработной платы.

Процент участия исполнителей в выполнении работ может быть рассчитан автоматически, с помощью кнопок над табличной частью с выбранными сотрудниками либо при помощи пунктов меню кнопки Еще:

  • Автоматическое распределение % участия – если процент участия для одного из исполнителей установлен вручную, то для остальных процент участия будет рассчитан автоматически;
  • Единые исполнители – будут установлены одинаковые исполнители для всех работ и установлены соответствующие проценты участия, согласно списку исполнителей текущей редактируемой работы;
  • Установить равные % – всем исполнителям будут установлены равные проценты участия в выполнении работ.

Табличная часть «Товары»

Кнопка Состояние может быть нажата или отжата. Если кнопка отжата, то вся таблица показывается черным шрифтом на белом фоне. Однако если кнопка нажата, то строки таблицы, соответствующие заказанным товарам, могут иметь различные цвета, показывающие состояние товара. Цвета имеют следующий смысл.

  • Зеленый – деталь имеется на складе в требуемом количестве.
  • Синий – деталь требуется заказать (детали в требуемом количестве нет).
  • Желтый – деталь заказана клиентом (деталь зарезервирована по заказу покупателя на эту деталь).
  • Фиолетовый – деталь получена по заказу клиента.

Табличная часть Товары содержит следующие дополнительные графы:

  • Количество. Количество деталей.
  • Ед., Коэффициент Единица измерения для данной детали и Коэффициент пересчета по отношению к базовой единице номенклатуры.

Если для документа выбрана хозяйственная операция План ремонта, то на форме документа появляется вкладка Планирование. На этой вкладке отображается план выполнения работ по заявке.

Ввод на основании

Документ Заявка на ремонт может быть введен на основании следующих документов:

  • Заказ на автомобиль,
  • Событие,
  • Дефектовочная ведомость,
  • Акт оценки автомобиля,
  • Заказ покупателя.

На основании документа Заявка на ремонт могут быть введены следующие документы:

  • Сводный ремонтный заказ,
  • Заказ-наряд,
  • Дефектовочная ведомость,
  • Заказ покупателя,
  • Заявка на аренду,
  • Снятие резервов заказов покупателя,
  • Событие,
  • Счет на оплату,
  • Жалоба клиента.

Печатные формы

  • Бланк заявки. Бланк заявки на проведение ремонтных работ с указанием заказчика и ремонтируемого автомобиля, а так же причины обращения заказчика. Для вывода в печатной форме схемы транспортного средства, отличной от используемой по умолчанию, в модели автомобиля (или в группе моделей) необходимо добавить картинку с данной схемой. Идентификатор картинки при этом должен быть "СхемаТС".
  • Рабочая заявка. Рабочая заявка на ремонт с перечнем работ и требующихся для их выполнения деталей.

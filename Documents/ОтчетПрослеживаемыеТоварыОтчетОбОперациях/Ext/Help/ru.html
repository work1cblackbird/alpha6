Документ "Отчет прослеживаемые товары отчет об операциях"

Назначение

Документ Отчет прослеживаемые товары отчет об операциях предназначен для формирования ежеквартального отчета организации для ФНС об операциях с товарами, подлежащими прослеживаемости. Для передачи отчета в ФНС формируется файл XML.

Относится к подсистеме «Прослеживаемые товары»

Реквизиты

  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ. 
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого формируется отчет. 
  • Подразделение компании. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в которой был выписан документ. 
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу.
  • Состояние. Реквизит является обязательным для заполнения. Ссылается на перечисление «Состояния уведомления прослеживаемости товаров». Текущее состояние документа устанавливается пользователем. При создании нового уведомления ему присваивается состояние "Новый". Если уведомление было отправлено в ФНС, то необходимо установить состояние "Отправлен в ФНС". 
  • Период отчета. Реквизит является обязательным для заполнения. Тип: «Дата». Дата начала квартала формирования отчета.
  • Номер корректировки. Тип: «Число». Номер текущей корректировки отчета.
  • Хозяйственная операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.

Табличная часть «Операции»

  • Дата операции. Тип: «Дата». Дата выполнения операции с прослеживаемым товаром.
  • Код вида операции. Ссылается на справочник «Коды операций прослеживаемости». Код текущей операции в соответствии с постановлением от ФНС.
  • Вид документа. Тип: «Число». Вид первичного документа. Устанавливается по следующим правилам: «1 – счет-фактура», «2 – корректировочный счет-фактура», «3 – универсальный передаточный документ», «4 – универсальный корректировочный документ» или «5 – иной первичный учетный документ».
  • Номер документа. Тип: «Строка». Номер первичного документа.
  • Дата документа. Тип: «Дата». Дата первичного документа. Для документа Заказ-наряд дата закрытия.
  • Наименование контрагента. Тип: «Строка». Наименование контрагента если он является юридическим лицом или ФИО для индивидуальных предпринимателей. Контрагент, который передал права собственности прослеживаемого товара или получил от организации.
  • ИНН контрагента. Тип: «Строка». ИНН контрагента юридического лица или индивидуального предпринимателя.
  • КПП контрагента. Тип: «Строка». Заполняется для контрагентов с формой собственность юридическое лицо.

 Табличная часть «Список РНПТ»

  • РНПТ. Тип: «Строка». Регистрационный номер партии товаров операции.
  • Код единицы измерения. Тип: «Строка». Код единицы измерения прослеживаемости по классификатору ОКЕИ.
  • Количество. Тип: «Число». Количество прослеживаемого товара в единицах прослеживаемости.
  • Сумма без НДС. Тип: «Число». Сумма прослеживаемого товара без учета налогов, в российских рублях с копейками.

При формировании xml файла проверяется заполнение необходимых реквизитов и их формат в соответствии с требованиями.

Печатные формы

  • Отчет об операциях с товарами, подлежащими прослеживаемости. Печатная форма, содержащая список операций с прослеживаемыми товарами, который был сформирован в документе.

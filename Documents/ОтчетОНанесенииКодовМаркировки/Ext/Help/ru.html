Документ "Отчет о нанесении кодов маркировки"

Назначение

Документ «Отчет о нанесении кодов маркировки» предназначен для формирования факта нанесения полученного кода маркировки на товар перед вводом ее в оборот. Используется не для всех товарных групп. 

ВАЖНО! Перед использованием документа необходимо убедится в наличии личного кабинета в ГИС МТ для данной организации. Необходимо указать в настройках способ обмена с ГИС МТ API. В файловом варианте необходимо вручную формировать отчет в личном кабинете Честного знака.

Относится к подсистеме Маркировка товаров раздела Снабжение.

Реквизиты

  • Статус. Реквизит является обязательным для заполнения. Ссылается на перечисление «Статусы документов маркировки». Отображает текущее состояние документа.
  • Товарная группа. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы маркировки». Определение товарной группы маркируемого товара.
  • Идентификатор документа. Тип значения: «Строка». Заполняется автоматически при отправке документа в ГИС МТ при использовании обмена через API.
  • Дата окончания срока годности. Реквизит является обязательным для заполнения для товарной группы «Молочная продукция». Тип значения: «Дата». Дата, после которой товар считается с истекшим сроком годности.
  • Были использованы на производстве. Тип значения: «Булево». Используется для товарной группы «Молочная продукция». Признак того, что товар был использован на производстве.
  • Строк годности меньше 72 часов. Тип значения: «Булево». Используется для товарной группы «Молочная продукция».  Признак того, что товар имеет срок годности меньше 72 часов. Это надо для определения строка годности при передачи в Честный знак через API. 
  • Идентификатор производственной линии. Реквизит является обязательным для заполнения для товарной группы «Табачная продукция». Тип значения: «Строка». Идентификатор производственного заказа. Используется для формирования запроса кодов маркировки в ГИС МТ.
  • Номер производственного заказа. Тип значения: «Строка». Используется для товарной группы «Табачная продукция». Номер производственного заказа.
  • Наименование бренда продукции. Тип значения: «Строка». Используется для товарной группы «Табачная продукция». Наименование бренда маркируемой продукции.
  • Идентификатор отчета о нанесении АСУТП. Тип значения: «Строка». Используется для товарной группы «Табачная продукция». Идентификатор отчёта о нанесении АСУТП.
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу. 
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ. На основании ИНН организации формируется запрос в ГИС МТ.  
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ. 
  • Документ основание. Ссылается на документ «Возврат от покупателя», «Чек», «Корректировка реализации». Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов. 
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.

Табличная часть «Товары»

  • Номенклатура. Реквизит обязательным для заполнения. Ссылается на справочник «Номенклатура». Номенклатура (товар).
  • Характеристика. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры, например, цвет, размер и т.п. 
  • Код маркировки. Реквизит является уникальным и обязательным для заполнения. Тип значения: «Строка». Код маркировки товара, которому изменяется состояние.

Кнопка Выгрузить используется для формирования документа «Возврат в оборот» и его отправки (при настройке обмена через API) в ГИС МТ. Для отображения факта отправки документа в учетную систему необходимо перевести документ в состояние «Отправлен». После успешной загрузки документа необходимо установить статус «Выполнен» и для перевода указанных кодов маркировки в новый статус в том случае, когда документ-основание не изменил.

Ввод на основании

Документ Возврат в оборот кодов маркировки может вводиться на основании следующих документов:

  • Заказ кодов маркировки.

Журналы, отображающие документ:

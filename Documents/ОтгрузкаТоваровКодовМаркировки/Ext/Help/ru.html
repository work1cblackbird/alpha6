Документ "Отгрузка товаров с выводом из оборота"

Назначение

Документ «Отгрузка товаров с выводом из оборота» предназначен для отображения факта отгрузки товара с кодами маркировки другому юридическому лицу с выводом из оборота по указанной причине и отправки документа в государственную информационную систему мониторинга за оборотом товаров (ГИС МТ). В остальных ситуациях информация о передаче товаров с кодами маркировки осуществляется через оператора ЭДО.

ВАЖНО! Перед использованием документа необходимо убедится в наличии личного кабинета в ГИС МТ для данной организации. Необходимо указать в настройках способ обмена с ГИС МТ через файловый обмен или API ГИС МТ.

ВАЖНО! При реализации товаров другому юридическому лицу информация о кодах маркировках передается в ГИС МТ через оператора ЭДО с отправкой электронного документа «Универсальный передаточный документ» (УПД). Если по какой-то причине не был сформирован УПД, то информацию об отгрузке товаров необходимо оформить данным документом для оповещения ГИС МТ об изменении статуса.

Относится к подсистеме Маркировка товаров раздела Снабжение.

Реквизиты

  • Статус. Реквизит является обязательным для заполнения. Ссылается на перечисление «Статусы документов маркировки». Отображает текущее состояние документа.
  • Товарная группа. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы маркировки». Определение товарной группы маркируемого товара.
  • Идентификатор документа. Тип значения: «Строка». Заполняется автоматически при отправке документа в ГИС МТ при использовании обмена через API.
  • Дата отгрузки. Реквизит является обязательным для заполнения. Тип значения: «Дата». Дата отгрузки товаров контрагенту.
  • Вид товарооборота. Реквизит является обязательным для заполнения. Ссылается на перечисление «Виды отгрузки товаров». Вид операции, по которой был передан товар контрагенту.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагенты и контакты». Контрагент, которому отгружен товар.
  • Отгрузка неучастнику. Тип значения: «Булево». Признак того, что указанный получатель товара зарегистрирован в ГИС МТ или на данный момент еще не зарегистрирован.
  • Дата первичного документа. Реквизит является обязательным для заполнения. Тип значения: «Дата». Дата документа, на основании которого производится отгрузка товаров.
  • Номер первичного документа. Реквизит является обязательным для заполнения. Тип значения: «Строка». Номер документа, на основании которого производится отгрузка товаров.
  • Вывод из оборота. Тип значения: «Булево». Признак того, что указанный указанные товары с кодами маркировки выводятся с оборота, то есть отгружается товар контрагенту не для дальнейшей перепродажи.
  • Дата вывода из оборота. Тип значения: «Дата». Дата вывода кодов маркировки из оборота. Реквизит является обязательным для заполнения при установке признака «Вывод из оборота».
  • Причина вывода. Ссылается на перечисление «Причины отгрузки товаров». Причина, по которой отгружаемый товар контрагенту выводится из оборота. Реквизит является обязательным для заполнения при установке признака «Вывод из оборота».
  • Идентификатор гос. контракта. Тип значения: «Строка». Идентификатор государственного контракта. Реквизит является обязательным для заполнения при указании причины вывода из оборота «Приобретение гос. предприятием».
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу. 
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ. На основании ИНН организации формируется запрос в ГИС МТ.  
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ. 
  • Документ основание. Ссылается на документ «Реализация товаров», «Заказ-наряд».  Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов. 
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.

Табличная часть «Товары»

  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Номенклатура (товар).
  • Характеристика. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры, например, цвет, размер и т.п. 
  • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество товаров с кодами маркировки.
  • Единица измерения. Реквизит является обязательным для заполнения. Ссылается на справочник «Единицы измерения». Количество кодов маркировки для заказа.
  • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент единицы измерения.
  • Цена за единицу с НДС. Тип значения: «Число». Цена за единицу товара с НДС.
  • Сумма. Тип значения: «Число». Сумма всегда равна Цена*Количество.
  • Ставка НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по номенклатурной позиции. 
  • Сумма НДС. Тип значения: «Число». Сумма НДС по номенклатурной позиции.

Табличная часть «Коды маркировки»

  • Код маркировки. Реквизит является уникальным и обязательным для заполнения. Тип значения: «Строка». Код маркировки товара, которому изменяется состояние.

Кнопка Выгрузить используется для формирования документа «Отгрузка товаров» и его отправки (при настройке обмена через API) в ГИС МТ. Для отображения факта отправки документа в учетную систему необходимо перевести документ в состояние «Отправлен». После успешной загрузки документа необходимо установить статус «Выполнен» и для перевода указанных кодов маркировки в новый статус в том случае, когда документ-основание не изменил.

Ввод на основании

Документ Отгрузка товаров с кодами маркировки может вводиться на основании следующих документов:

  • Реализация товаров,
  • Заказ-наряд.

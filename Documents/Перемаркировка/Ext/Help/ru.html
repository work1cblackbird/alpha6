Документ "Перемаркировка"

Назначение

Документ «Перемаркировка» предназначен для формирования факта перемаркировки кодов маркировки и отправки документа в государственную информационную систему мониторинга за оборотом товаров (ГИС МТ). Данный документ используется для отображения таких операций как повторная маркировка товаров без вывода товаров из оборота и повторная маркировка товаров, раннее выведенных из оборота, с повреждённым средством идентификации или без него.

ВАЖНО! Перед использованием документа необходимо убедится в наличии личного кабинета в ГИС МТ для данной организации. Необходимо указать в настройках способ обмена с ГИС МТ через файловый обмен или API ГИС МТ.

Относится к подсистеме Маркировка товаров раздела Снабжение.

Реквизиты

  • Статус. Реквизит является обязательным для заполнения. Ссылается на перечисление «Статусы документов маркировки». Отображает текущее состояние документа.
  • Товарная группа. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы маркировки». Определение товарной группы маркируемого товара.
  • Идентификатор документа. Тип значения: «Строка». Заполняется автоматически при отправке документа в ГИС МТ при использовании обмена через API.
  • Причина. Реквизит является обязательным для заполнения. Ссылается на перечисление «Причины перемаркировки». Причина перемаркировки товара. 
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ. На основании ИНН организации формируется запрос в ГИС МТ.  
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ. 
  • Документ основание. Ссылается на документ «Возврат от покупателя», «Чек», «Корректировка реализации», «Списание кодов маркировки». Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов. 
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.

Табличная часть «Товары»

  • Номенклатура. Реквизит и обязательным для заполнения. Ссылается на справочник «Номенклатура». Предыдущая номенклатура (товар). Является обязательным для заполнения для причины «Выявлены ошибки описания товара».
  • Характеристика. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры, например, цвет, размер и т.п. 
  • Код маркировки. Реквизит является уникальным для заполнения. Тип значения: «Строка». Предыдущий код маркировки товара. Является обязательным для заполнения для причины «Выявлены ошибки описания товара».
  • Номенклатура новая. Реквизит является обязательным для заполнения. Ссылается на справочник «Номенклатура». Текущая номенклатура (товар).
  • Характеристика новая. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры, например, цвет, размер и т.п. 
  • Код маркировки новый. Реквизит является уникальным и обязательным для заполнения. Тип значения: «Строка». Текущий код маркировки товара.
  • Код ТН ВЭД. Реквизит является обязательным для заполнения. Ссылается на справочник «Классификатор ТН ВЭД». Код товара по классификатору ТН ВЭД. 
  • Страна производства. Ссылается на справочник «Страны мира». Страна, в которой был произведен товар. Реквизит является обязательным для заполнения для товарной группы «Шины и покрышки пневматические резиновые новые».
  • Цвет. Тип значения: «Строка». Цвет товара. Реквизит является обязательным для заполнения для товарной группы «Предметы одежды, белье постельное, столовое, туалетное и кухонное».
  • Размер. Тип значения: «Строка». Размер товара. Реквизит является обязательным для заполнения для товарной группы «Предметы одежды, белье постельное, столовое, туалетное и кухонное».
  • Причина перемаркировки. Реквизит является обязательным для заполнения. Ссылается на перечисление «Причины перемаркировки». Причина перемаркировки товара.
  • Оплачен. Тип значения: «Строка». Признак того, что товар был оплачен покупателем. Реквизит является обязательным для заполнения для причины «Возврат товаров с поврежденным СИ/без СИ при розничной реализации».
  • Дата повторной маркировки. Реквизит является обязательным для заполнения. Тип значения: «Дата». Дата перемаркировки товара.
  • Вид документа соответствия. Ссылается на перечисление «Виды документов соответствия». Вид документа, которым подтверждено соответствие товара.
  • Номер документа соответствия. Тип значения: «Строка». Номер указанного документа соответствия.
  • Дата документа соответствия. Тип значения: «Дата». Дата указанного документа соответствия.
  • Вид первичного документа. Реквизит является обязательным для заполнения. Ссылается на перечисление «Виды первичных документов». Вид документа, на основании которого был возвращен товар. Реквизит является обязательным для заполнения для причины «Возврат товаров с поврежденным СИ/без СИ при розничной реализации».
  • Наименование первичного документа.  Тип значения: «Строка». Наименование документа, на основании которого был возвращен товар. Реквизит является обязательным для заполнения для причины «Возврат товаров с поврежденным СИ/без СИ при розничной реализации».
  • Дата первичного документа. Тип значения: «Дата». Дата документа, на основании которого был возвращен товар. Реквизит является обязательным для заполнения для причины «Возврат товаров с поврежденным СИ/без СИ при розничной реализации».
  • Номер первичного документа. Тип значения: «Строка». Номер документа, на основании которого был возвращен товар. Реквизит является обязательным для заполнения для причины «Возврат товаров с поврежденным СИ/без СИ при розничной реализации».

Кнопка Выгрузить используется для формирования документа «Перемаркировка» и его отправки (при настройке обмена через API) в ГИС МТ. Для отображения факта отправки документа в учетную систему необходимо перевести документ в состояние «Отправлен». После успешной загрузки документа необходимо установить статус «Выполнен» и для перевода указанных кодов маркировки в новый статус.

В подменю Заполнение командной панели табличной части документа добавлена дополнительная команда Заполнить коды маркировок по заказам на эмиссию. Заполняет новыми кодами маркировки табличную часть, которые были получены по введенными на основании документа заказам кодов на маркировку.

Ввод на основании

Документ Перемаркировка может вводиться на основании следующих документов:

  • Возврат от покупателя,
  • Чек,
  • Корректировка реализации,
  • Списание кодов маркировки.
Документы, вводимые на основании:
  • Заказ кодов маркировки.

Журналы, отображающие документ:

  • Документы кодов маркировки

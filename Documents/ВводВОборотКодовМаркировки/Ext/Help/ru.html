Документ "Ввод в оборот кодов маркировки"

Назначение

Документ «Ввод в оборот кодов маркировки» предназначен для формирования факта ввода в оборот кодов маркировки и отправки документа в государственную информационную систему мониторинга за оборотом товаров (ГИС МТ). 

ВАЖНО! Перед использованием документа необходимо убедится в наличии личного кабинета в ГИС МТ для данной организации. Необходимо указать в настройках способ обмена с ГИС МТ через файловый обмен или API ГИС МТ.

Относится к подсистеме Маркировка товаров раздела Снабжение.

Реквизиты

  • Статус. Реквизит является обязательным для заполнения. Ссылается на перечисление «Статусы документов маркировки». Отображает текущее состояние документа.
  • Товарная группа. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы маркировки».. Определение товарной группы маркируемого товара.
  • Идентификатор документа. Тип значения: «Строка». Заполняется автоматически при отправке документа в ГИС МТ при использовании обмена через API.
  • Вид документа. Реквизит является обязательным для заполнения. Ссылается на перечисление «Способы ввода в оборот». Способ ввода в оборот товара.
  • Ответственный. Ссылается на справочник «Сотрудники». Сотрудник, который является ответственным за документ.
  • Дата получения товара. Тип значения: «Дата». Реквизит является обязательным для заполнения при указании вида документа «Получение от физических лиц». Дата поступления маркируемого товара.
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу. 
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ. На основании ИНН организации формируется запрос в ГИС МТ.  
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ. 
  • Документ основание. Ссылается на документ «Заказ кодов маркировки».  Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов. 
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.

Табличная часть «Товары»

  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Номенклатура (товар).
  • Характеристика. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры, например, цвет, размер и т.п. 
  • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество товаров с кодами маркировки.
  • Единица измерения. Реквизит является обязательным для заполнения. Ссылается на справочник «Единицы измерения». Количество кодов маркировки для заказа.
  • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент единицы измерения.
  • Страна производства. Ссылается на справочник «Страны мира». Страна, в которой был произведен товар.
  • Вид документа соответствия. Ссылается на перечисление «Виды документов соответствия». Вид документа, которым подтверждено соответствие товара.
  • Номер документа соответствия. Тип значения: «Строка». Номер указанного документа соответствия.
  • Дата документа соответствия. Тип значения: «Дата». Дата указанного документа соответствия.
  • Дата получения товара. Тип значения: «Дата». Заполняется при указании вида документа «Получение от физических лиц». Дата поступления маркируемого товара.

Табличная часть «Коды маркировки»

  • Код маркировки. Реквизит является уникальным и обязательным для заполнения. Тип значения: «Строка». Код маркировки товара, которому изменяется состояние.

Кнопка Выгрузить используется для формирования документа «Ввод в оборот» и его отправки (при настройке обмена через API) в ГИС МТ. Для отображения факта отправки документа в учетную систему необходимо перевести документ в состояние «Отправлен». После успешной загрузки документа необходимо установить статус «Выполнен» для перевода указанных кодов маркировки в новый статус.

В подменю Заполнение командной панели табличной части документа добавлены дополнительные команды:
  • Заполнить коды маркировок по заказам на эмиссию. Заполняет кодами маркировки табличную часть, которые были получены по введенными на основании документа заказам кодов на маркировку.
  • Заполнить складскими остатками. Заполняет табличную часть документа «Товары» остатками маркируемого товара на выбранном складе.
  • Заполнить складским остаткам немаркированных товаров. Заполняет табличную часть документа «Товары» остатками маркируемого товара на выбранном складе с учетом товаров, которым ранее был введен в оборот код маркировки.
  • Заполнить остатками по группе номенклатуры. Заполняет табличную часть документа «Товары» остатками маркируемого товара на выбранном складе и указанной группе номенклатуры.

Документы, вводимые на основании:

Произвольная форма "Заполнение по документу основанию"

Назначение

Произвольная форма документа «Бюджет закупок автомобилей», предназначена для заполнения документа данными, на основании  документа «Бюджет продаж автомобилей» или документа «Бюджет закупок автомобилей». Позволяет вводить бюджет моделей на основании бюджета моделей, бюджет вариантов комплектации на основании бюджета вариантов комплектации, а также бюджет вариантов комплектации на основании бюджета моделей. В последнем случае предлагается выполнить распределение сумм документа основания по позициям вариантов комплектации,  которые в свою очередь заполняются методами наивного прогнозирования, либо вручную.

Способы вызова

Форма вызывается из меню командной панели табличной части документа «Бюджет закупок автомобилей»: "Заполнение по документу основанию", а также в случае ввода на основании штатными средствами платформы 1С.

Принцип работы с формой

В случае если заполняемый документ имеет вид бюджета "Бюджет закупок по моделям", то вся работа с формой сводится к выбору документа основания. В этом случае в качестве документа основания может выступать документ «Бюджет продаж автомобилей» или «Бюджет закупок автомобилей»  с аналитикой по моделям, причем периодичности сценариев планирования документов должны совпадать. Также должно выполняться условие, что период планирования документа основания должен быть не больше периода планирования заполняемого документа. Далее таблица формы заполнится табличной частью документа основания и будет недоступна для редактирования. При нажатии "ОК" данные таблицы скопируются в табличную часть заполняемого документа.

Если происходит заполнением бюджета закупок с аналитикой по вариантам комплектации ("Бюджет закупок по вариантам комплектации"), то в качестве основания может выступать документ «Бюджет продаж автомобилей»  или документ «Бюджет закупок автомобилей» с любой аналитикой, при этом накладываются те же ограничения на периодичность сценариев планирования и дату планирования документа основания. Если в качестве документа основания выбран бюджет с аналитикой по вариантам комплектации, то дальнейшая работа с формой будет аналогично предыдущему случаю. Если же выбран бюджет с аналитикой по моделям, то дальнейшая работа с формой будет иной. В этом случае таблица на форме будет отображаться в виде дерева значений. На первом уровне будут находиться данные по моделям (заполнятся по табличной части документа основания). Второй уровень будет содержать данные по вариантам комплектации (заполняется вручную, либо при помощи заполнения наивным прогнозированием - кнопка "Заполнить" на командной панели). Варианты комплектации добавляется строго в свою "ветку" с учетом моделей, к которым они принадлежит. В итоге мы имеем двухуровневую таблицу. В колонке "Сумма базовая" отображается сумма модели, в колонке "Сумма (упр)" содержатся суммы вариантов комплектации. Далее необходимо распределить суммы моделей по суммам вариантов комплектации (подогнать суммы вариантов комплектации таким образом, чтобы итоговые значения "Сумма упр" по каждой модели максимально соответствовал значению базовой суммы). Распределение производится по весовым коэффициентам, которые могут рассчитываться тремя способами: по количеству варианта комплектации, по сумме варианта комплектации, по маржинальной сумме варианта комплектации (реквизит "Метод распределения сумм моделей"). Также можно выбрать показатель, который будет изменяться в ходе "подгонки" сумм: Количество или Цена вариантов комплектации.

Реквизиты

  • Документ основание. Ссылка на документ «Бюджет продаж автомобилей» или Ссылка на документ «Бюджет закупок автомобилей».
  • Смещение планирования. Число. Количество отступаемых периодов от планируемого периода.  Используется при вызове формы наивного прогнозирования.
  • Количество периодов. Число. Количество анализируемых периодов.  Используется при вызове формы наивного прогнозирования.
  • Количество сезонов. Число. Количество анализируемых сезонов.  Используется при вызове формы наивного прогнозирования.
  • Коэффициент роста. Число. Экспертная оценка роста объема закупок в прогнозируемом периоде.  Используется при вызове формы наивного прогнозирования.
  • Коэффициент сезонности. Число. Экспертная оценка сезонной составляющей.  Используется при вызове формы наивного прогнозирования.
  • Показатель планирования. Перечисление  «Типы анализа». Вид прогнозируемого ресурса (количество или сумма упр.)  Используется при вызове формы наивного прогнозирования.
  • Способ округления. Число. Определяет способ округления ресурса "количество"
  • Сценарий планирования. Справочник  «Сценарий планирования». Соответствует сценарию планирования документа  «Бюджет закупок автомобилей», из которого была вызвана форма. В форме не редактируется. 
  • Тип анализа. Перечисление  «Типы анализа». Используется при вызове формы наивного прогнозирования.
  • Метод распределения моделей. Перечисление  «Методы распределения моделей» . Способ расчета весовых коэффициентов вариантов комплектации.
  • Параметр управления распределением. Булево. Определяет ресурс, который будет изменяться в ходе распределения (количество, либо цена)  

 

Дополнительная информация

В случае если происходит распределение сумм моделей, и в качестве изменяемого показателя выбрано "количество", то точной "подгонки" в общем случае не получится.

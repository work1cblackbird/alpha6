Документ "Заказ-наряд"

Назначение

Документ Заказ-наряд предназначен для фиксации факта выполнения ремонта автомобиля, а также для учета запчастей, использованных в процессе ремонта автомобиля. В процессе выполнения ремонтных работ документ находится в не проведенном состоянии и должен проводиться только по завершению выполнения ремонта. Относится к подсистеме Автосервис раздела Автосервис.

При проведении документ выполняет следующие действия:

  • списывает кредиторскую задолженность компании перед плательщиком по сделке заказ-наряда или начисляет дебиторскую задолженность плательщика перед компанией на стоимость работ и использованных запчастей;
  • списывает из цеха использованные при выполнении работ запасные части;
  • фиксирует информацию о выработке исполнителей по данному заказ-наряду.

Закрытие и проведение заказ-наряда возможно, если только в производство перемещены все детали, необходимые для выполнения работ.

Реквизиты

  • Заказчик. Контрагент: владелец ремонтируемого автомобиля.
  • Автомобиль. Ремонтируемый автомобиль. Справа отображается гиперссылка, по которой открывается отчет История автомобилей, сформированный для данного автомобиля.
  • Ремонт. Выбирается из списка, который соответствует справочнику Виды ремонта. В зависимости от вида ремонт может быть платным или бесплатным. В соответствии со статьями доходов и расходов, указанных в справочнике Виды ремонта для каждого вида платного или бесплатного ремонта, будут распределены доходы и расходы. При изменении реквизита изменяться нормочас в соответствии с нормочасом в ремонте. Следует учесть то, что нормочас будет сначала взят из договора взаиморасчетов, а если там нормочас пустой, то уже из ремонта.  
  • Цех. Основной цех, в котором происходит ремонт.
  • Мастер. Сотрудник, ответственный за выполнение ремонта.
  • Диспетчер. Сотрудник, принявший заказ.
  • Сводный ремонтный заказ. Документ Сводный ремонтный заказ, к которому относится данный заказ-наряд.
  • Способ зачета авансов. Если установлено в значение "Автоматически", то при проведении документа все незакрытые сделки будут автоматически закрываться (в порядке ФИФО). В значении "Не зачитывать" - незакрытые сделки не будут погашены. Для договоров с признаком учета взаиморасчетов "По расчетным документам" доступен вариант "По документу" с возможностью выбора авансовых документов для зачета оплаты.
  • Кампания. Сервисная кампания, в рамках которой осуществляется ремонт.
  • Причины обращения. Текстовое поле. Причина обращения заказчика в автосервис. Пакет работ.
  • Состояние. Текущее состояние заказ-наряда. Выбирается из справочника Состояния заказ-нарядов.
  • Маркетинговая программа. Маркетинговая программа, в рамках которой осуществляется ремонт. Выбирается из справочника Маркетинговые программы.
  • Пробег ТО. Пробег автомобиля между техобслуживаниями.
  • Дата. Дата следующего техобслуживания.
  • Склад компании. Используется только для вида ремонта с типом «Сборка автомобиля». Склад компании, на который будет оприходован собранный автомобиль.

Кнопка Оплатить через кассу командной панели диалогового окна Заказ-наряд позволяет ввести документ Чек на оплату.

Кнопка Ввести на основании имеет добавочный пункт меню Извлечение товаров из производства. Если документ Заказ-наряд не проведен, то при выборе данного пункта открывается форма Извлечение товаров из производства. В этом диалоговом окне мы имеем возможность выбрать номенклатуру для извлечения из производства и указать, на какой склад ее нужно переместить.

Табличная часть «Автоработы»

Вкладка Автоработы содержит две таблицы – список пакетов работ (причин обращений в автосервис) и список авторабот, соответствующих данному ремонту. Дополнительные графы таблицы авторабот:

  • Авторабота. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник Автоработы. Работа для выполнения заказ-наряда.
  • Количество. Реквизит является обязательным для заполнения. В этой графе требуется указать, сколько раз требуется выполнить указанную работу. Например, если работа «Балансировка колеса» выполняется для четырех колес, в графе Количество следует указать число 4.
  • н/ч (норма времени). Нормочас и норма времени. В графе н/ч из справочника Нормочасы выбирается вид нормочаса, соответствующий стоимости одного часа выполнения указанной работы. В колонке н/ч указывается нормативное количество часов, необходимое для выполнения работы для данного класса автомобиля. Если требуется указать стоимость работы не в нормочасах, а в денежном выражении, то в колонке н/ч требуется выбрать нормочас, соответствующий валюте цены работы (для этого нормочаса в справочнике Нормочасы значение реквизита Цена нормочаса должно быть установлено равным 1). В колонке н/ч при этом должна быть указана стоимость выполнения указанной работы в валюте нормочаса.
  • Цена. Цена выполнения указанной работы (нормочаса) в валюте документа.
  • Сумма. Цена выполнения указанной работы, умноженная на число, указанное в графе Количество. Если расчет работы ведется по нормочасам, то дополнительно производится умножение результата на значение в графе Норма времени.
  • Материалы. Материалы, необходимые для выполнения работы.
  • Субподряд. Контрагент, с которым ведутся взаиморасчеты по субподрядам. Если какие-либо работы выполняет субподрядчик, то на основании документа Заказ-наряд вводится документ Поступление товаров (с хозяйственной операцией Услуги по субподряду), в котором указываются данные об услуге, предоставляемой субподрядчиком.
  • Способ расчета стоимости работы. Указывает каким образом будет произведен расчет суммы автоработы. Если установлен признак "Фиксированной суммой", то сумма автоработы рассчитывается по формуле Количество * Цена. Иначе - Количество * Норма времени * Цена
  • Исполнители. В графе Исполнители содержится список исполнителей, которые выполняют работы по данному заказу. Исполнителями авторабот могут быть только сотрудники, у которых в их карточках (элементах справочника Сотрудники) установлен флажок Исполнитель. Для закрытия заказ-наряда требуется, чтобы общий процент участия исполнителей в каждой работе составлял 100%. Процент участия исполнителей в выполнении работ может быть рассчитан автоматически, с помощью команд формы подбора исполнителей:
    • Автоматическое распределение % участия – если процент участия для одного из исполнителей установлен вручную, то для остальных процент участия будет рассчитан автоматически;
    • Единые исполнители – будут установлены одинаковые исполнители для всех работ и установлены соответствующие проценты участия, согласно списку исполнителей текущей редактируемой работы;
    • Установить равные % – всем исполнителям будут установлены равные проценты участия в выполнении работ.

Табличная часть «Товары»

Вкладка Товары содержит список запчастей, необходимых для выполнения авторабот. При этом детали могут быть в наличии на предприятии, заказаны у поставщика.

Кнопка Состояние может быть нажата или отжата. Если кнопка отжата, то вся таблица показывается черным шрифтом на белом фоне. Однако если кнопка нажата, то строки таблицы, соответствующие заказанным товарам, могут иметь различные цвета, показывающие состояние товара. Цвета имеют следующий смысл.

  • Зеленый – деталь имеется на складе в требуемом количестве.
  • Синий – деталь требуется заказать (детали в требуемом количестве нет).
  • Желтый – деталь заказана клиентом (деталь зарезервирована по заказу покупателя на эту деталь).
  • Фиолетовый – деталь получена по заказу клиента.
  • Серый – деталь отгружена в производство в полном объеме.

Начертание шрифта «полужирный» актуален для деталей, которые не отгружены в производство полностью.

Кнопка Списание товаров содержит три варианта списания материалов по документу:

  • На себестоимость. В случае если флаг установлен, то стоимость товаров войдет в себестоимость автомобиля, выбранного в "Заказ-наряде".
  • По себестоимости. Это означает, что стоимость запчастей в "Заказ-наряде" будет рассчитана по себестоимости списанных товаров (а не по "Типу цен"). Чаще всего это используется для внутренних "Заказ-нарядов", там, где не нужна наценка. Расчет выполняется в момент перевода в статус "Выполнен", когда все запчасти выданы в производство. Работает совместно с включенным правом "Управление признаком списания товаров по себестоимости в заказ-наряде"-41384, если право установлено, то аналогичный флаг можно установить в "Виде ремонта. Из вида ремонта флаг попадает в "Заказ-наряд". Если право включено - то появляется возможность  изменить признак в "Заказ-наряде." 
  • По Заказ-нарядам. Если флаг установлен, то программа не будет выбирать все открытые "Заказы покупателя" контрагента и списывать их в порядке "ФИФО", а будут списываться именно те "Заказы покупателя", которые связаны с "Заказ-нарядом".

Меню кнопки Заполнение вкладки Товары, кроме обычных возможностей, содержит дополнительные пункты:

  • Заполнить деталями из производства. Позволяет заполнить табличную часть вкладки Товары деталями, на данный момент уже переданными в производство. При этом если какая-либо деталь отсутствует в таблице товаров, то эта деталь будет добавлена в том количестве, в каком она была передана в производство. Если такая деталь уже имеется в таблице товаров, то для этой детали будет установлено количество, не меньшее, чем числится в производстве.
  • Заполнить заказанными деталями. Табличная часть будет заполнена деталями, заказанными по заказам покупателя, введенным на основании заказ-наряда.

Меню кнопки Действие вкладки Товары содержит следующие пункты:

  • Заказать. Формируется заказ поставщику на детали по заказ-наряду.
  • Зарезервировать. Резервирование деталей по заказ-наряду на складе.
  • В производство. Формирует перемещения в производство. Автоматически формирует соответствующий документ для всех табличных строк, соответствующих имеющимся деталям. Если настройка Автоматическое формирование перемещений в производство для подразделения установлена, то перемещение деталей в производство будет производиться автоматически при каждой записи документа Заказ-наряд. Если для пользователя не установлено право Перемещение деталей в производство, то документы перемещения будут формироваться, но не будут проводиться.
  • Из производства. Перемещение деталей в производстве по заказ-наряду обратно на склад.

Если перемещения были проведены, то решение запрещает проведение ошибочного заказ-наряда. Требуется предварительно извлечь товары из производства.

При изменении количества деталей в заказ-нарядах осуществляется контроль количества данных деталей в производстве: количество деталей в документе не должно быть меньше суммы деталей в производстве, заказанного количества и зарезервированного количества деталей. Иначе пользователю выводится служебное сообщение. Данный контроль регулируется правом Контроль количества деталей в производстве.

Для того, чтоб стал активен способ списания товаров "На себестоимость" необходим вид ремонта - бесплатный и не "Комплектация".

В решении «Альфа-Авто: Автосалон+Автосервис+Автозапчасти. КОРП. Редакция 6» предусмотрено разграничение прав пользователя на редактирование списка деталей на вкладке Товары (право Редактирование деталей заказ-наряда) и на редактирование перечня работ и списка исполнителей этих работ на вкладке Автоработы (право Редактирование работ заказ-наряда).

При нажатии на кнопку Информация об остатках на экран выводится отчет Расшифровка информации об остатках.

Под таблицей списка авторабот отображаются пиктограммы состояния материалов автосервиса с указанием количества соответствующего материала. Расшифровка пиктограмм представлена на следующем рисунке.


 Пиктограммы состояния материалов автосервиса:
Данная пиктограмма обозначает количество материала на складе;
Данная пиктограмма обозначает количество заказанного материала;
Данная пиктограмма обозначает количество материала в резерве;
Данная пиктограмма обозначает количество материала в производстве;
Данная пиктограмма обозначает количество материала, которое осталось переместить в производство;
Данная пиктограмма обозначает количество материала, которое возможно переместить в производство;
С помощью данной пиктограммы можно обновить показатели состояния материалов автосервиса.

Табличная часть «Материалы заказчика»

Вкладка Материалы заказчика заказ-наряда содержит данные о материалах, предоставленных заказчиком для ремонта автомобиля.

Табличная часть «Зачет авансов»

Доступно только для платного вида ремонтов. Вкладка Зачет авансов содержит способ зачета оплаты/авансов, а так же список подобранных вручную документов оплаты и суммы оплаты для погашения задолженности. Так же в список можно добавить документы с помощью кнопки Подбор авансов. Данный список документов оплат доступен только при выборе способа зачета оплаты/авансов "По документу".
  • Документ аванса. Реквизит является уникальным и обязательным для заполнения. Документ, на основании которого будет погашена задолженность.
  • Сумма. Реквизит обязательным для заполнения. Сумма для зачета по документу аванса.

Вкладка «Дополнительно»

На вкладке Дополнительно содержатся поля с рекомендациями для владельца автомобиля, описание гарантий на выполненные работы и примененные детали. Также здесь отображаются поля дополнительных реквизитов, заданных для документа.

Для ввода документа Приходный кассовый ордер на основании заказ-наряда требуется, чтобы для пользователя было установлено право Ввод оплаты по заказ-наряду.

Ввод на основании

Документ Заказ-наряд вводится на основании документов и справочников:

  • Заявка на ремонт,
  • Сводный ремонтный заказ,
  • Заказ покупателя,
  • Автомобили,
  • Контрагенты и контакты.

Документы, вводимые на основании:

  • Заказ покупателя,
  • Приходный кассовый ордер,
  • Перемещение товаров в производство,
  • Извлечение товаров из производства,
  • Расходный складской ордер на автомобили,
  • Заказ внутренний,
  • Снятие резервов заказов покупателя,
  • Перемещение незавершенного производства,
  • Жалоба клиента,
  • Заказ поставщику,
  • Разукомплектация автомобилей,
  • Приходный складской ордер на автомобили,
  • Корректировка реализации,
  • Заявка на хранение шин,
  • Списание товаров,
  • Комплектация,
  • Возврат от покупателя,
  • Заказ на автомобиль,
  • Корректировка движений,
  • Чек на оплату,
  • Акт разногласий,
  • Поступление товаров,
  • Переразмещение автомобилей,
  • Счет на оплату,
  • Счет-фактура выданный,
  • Ввод остатков товаров в производстве.

Печатные формы

Состав печатаемых форм зависит от права Печать расширенного пакета документов заказ-наряда.

Если это право установлено, то можно печатать следующие печатные
формы:

  • Заявка предварительная;
  • Акт осмотра;
  • Бланк заявки;
  • Заказ-наряд (договор);
  • Акт приема;
  • Сопроводительный лист;
  • Заявка на детали;
  • Квитанция к заказ-наряду*;
  • Акт сдачи;
  • ТОРГ-12 (Товарная накладная);
  • Акт об оказании услуг;
  • Акт об оказании услуг (услуги+материалы);
  • Универсальный передаточный документ;
  • Заказ-наряд (шаблон).

Если это право отключено, то можно печатать следующие печатные формы:

  • Бланк заявки;
  • Рабочая заявка;
  • Приемный акт;
  • Заказ-наряд*;
  • Акт сдачи;
  • ТОРГ-12 (Товарная накладная);
  • Акт об оказании услуг;
  • Акт об оказании услуг (услуги+материалы);
  • Универсальный передаточный документ;

  • Заказ-наряд (шаблон).

Печатные формы, отмеченные знаком «*», не могут быть сформированы при следующих условиях:

  1. Если заказ-наряд не закрыт и у пользователя не установлено право Печать заказ-наряда без закрытия.
  2. Если у данного контрагента есть другие открытые заказ-наряды, а у пользователя не установлено право Печать заказ-наряда при наличии других открытых заказ-нарядов.
